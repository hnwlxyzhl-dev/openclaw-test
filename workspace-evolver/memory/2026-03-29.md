### 关键新知识点

#### Anthropic 多智能体模式（权威来源）

**5 种 Workflow 模式**：
1. **Prompt Chaining** - 顺序步骤，每步输出是下一步输入
2. **Routing** - 分类输入，路由到专门处理
3. **Parallelization** - 并行执行子任务或投票
4. **Orchestrator-Workers** - 编排者动态分解任务，工作者执行
5. **Evaluator-Optimizer** - 生成者+评估者循环迭代

**Autonomous Agents**：
- 从用户指令开始，独立规划和操作
- 适合开放式、不可预测步骤的任务
- 关键：简单设计、透明规划、精心设计 ACI

#### OpenClaw 多智能体实现

**路由机制**：
- 按 peer（用户/群组）路由
- 按 accountId（多账号）路由
- 按 channel（渠道）路由
- 优先级：peer > guildId > teamId > accountId > default

**子智能体**：
- `sessions_spawn` 启动子智能体
- 支持嵌套（maxSpawnDepth: 2 = 编排者模式）
- 自动 announce 结果回主会话

#### AI 框架对比

| **LangGraph** | 有状态工作流，精确控制 |

| **AutoGen** | 微软出品，多智能体对话 |
| **CrewAI** | 角色扮演，企业级部署 |
| **DSPy** | 声明式 Prompt 优化 |

#### RAG 范式演进

1. **Naive RAG** - 简单检索+生成（低精度/召回）
2. **Advanced RAG** - 优化检索质量（预检索/检索/后检索）
3. **Modular RAG** - 模块化（Search/Memory/Fusion/Routing）

#### RAG 优化技术

- **混合检索**：关键词 + 语义
- **递归检索**：小块 → 大块
- **StepBack Prompting**：抽象推理
- **子查询**：分解复杂问题
- **HyDE**：生成假设文档检索
- **重排序**：优化结果顺序

#### RAG vs Fine-tuning

- **RAG**：实时知识，低成本，高可解释性
- **Fine-tuning**：格式/风格定制，高成本
- **组合**：RAG 提供知识，Fine-tuning 优化格式

#### RAG 评估

- **质量评分**：上下文相关性、答案忠实度、答案相关性
- **能力评估**：噪声鲁棒性、负面拒绝、信息整合、反事实鲁棒性
- **工具**：RAGAS, ARES, TruLens, RGB, RECALL
