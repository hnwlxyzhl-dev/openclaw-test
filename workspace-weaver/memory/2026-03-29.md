**核心原则（4 大）：**

1. **逻辑清晰**（Logical Clarity）
   - 金字塔原理：结论先行，层层递进
   - MECE 原则：相互独立，完全穷尽
   - 信息架构：层次分明，结构合理

2. **结构清晰**（Structural Clarity）
   - 标题层次：H1 唯一，不超过 4 级
   - 段落组织：3-5 句为宜，主题明确
   - 信息架构：分类清晰，导航方便

3. **图文并茂**（Visual Richness）
   - 代码块：指定语言，添加注释，可运行
   - 表格：对比数据，格式统一，对齐正确
   - 图片：视觉说明，质量高，说明清晰
   - emoji：情感标识，视觉增强

4. **清晰表达**（Clear Expression）
   - 简洁性：一句话一个观点，3-5 句/段
   - 准确性：用词精确，数据准确，逻辑严密
   - 可读性：段落简短，留白充足，重点突出
   - 完整性：信息完整，上下文清晰，前后呼应

**高级技巧（8 个）：**

1. **标题层次** - H1 唯一，层次 ≤ 4，逻辑清晰
2. **段落结构** - 主题句 + 支持句 + 总结句
3. **列表使用** - 无序（并列）+ 有序（顺序）+ 任务（待办）
4. **代码示例** - 指定语言，添加注释，可运行，有说明
5. **表格设计** - 对比数据，对齐正确，格式统一
6. **引用强调** - 粗体（重要）+ 斜体（术语）+ 代码（命令）
7. **链接图片** - 描述清晰，链接有效，图片有说明
8. **视觉元素** - emoji + 分隔线 + 引用块 + 折叠内容

**质量标准（4 维 100 分）：**

1. **结构清晰度**（0-100）
   - 标题层次合理（0-25）

**写作流程（3 步）：**

1. **准备阶段**（20% 时间）
   - 确定主题和目标读者
   - 收集素材和参考资料
   - 构建大纲和标题层次

2. **撰写阶段**（60% 时间）
   - 快速初稿，不要追求完美
   - 逐节完善，添加示例
   - 插入图表，优化格式

3. **优化阶段**（20% 时间）
   - 内容检查：逻辑、结构、信息、表达
   - 格式检查：标题、列表、代码、链接
   - 最终校对：术语、错别字、一致性

**常见错误（4 类）：**
- ❌ 标题层次混乱（跳级、多个 H1）
- ❌ 段落过长（超过 5 句）
- ❌ 缺少视觉元素（纯文字）
- ❌ 术语不一致（同一概念多种叫法）

**模板系统（3 类）：**
1. **技术文档** - 概述 + 特性 + API + FAQ + 更新日志
2. **学习笔记** - 目标 + 概念 + 收获 + 疑问 + 行动
3. **项目文档** - 概述 + 需求 + 设计 + 开发 + 测试

**专家级秘诀（4 个）：**
1. **一致性 > 完美** - 术语、格式、风格保持一致
2. **简洁 > 详细** - 3-5 句段落，15-25 字句子
3. **示例 > 描述** - 代码示例 > 文字描述
4. **结构 > 内容** - 清晰结构 > 丰富内容

## 💡 核心知识点总结

### **Stable Diffusion vs Diffusion Model：**

**Diffusion Model（广义）：**
- 操作空间：像素空间（512×512×3）
- 维度：786,432
- 计算成本：高
- 代表：DDPM, DDIM

**Stable Diffusion（特指）：**
- 操作空间：潜在空间（64×64×4）
- 维度：16,384
- 计算成本：低（1/64）
- 核心创新：VAE降维 + CLIP文本编码

---

### **三大核心组件：**

**1. CLIP Text Encoder：**
- 输入：文本（最多77 token）
- 处理：12层Transformer
- 输出：[77, 768] 向量
- 参数：123M

**2. VAE：**
- Encoder：512×512 → 64×64（降维64倍）
- Decoder：64×64 → 512×512（重建）
- 参数：83M（Enc 34M + Dec 49M）

**3. U-Net：**
- 结构：Encoder-Bottleneck-Decoder
- Skip Connections：连接编码器和解码器
- Cross-Attention：融合文本条件
- Time Embedding：编码时间步
- 参数：860M（主要计算成本）

---

### **训练过程（6步）：**

1. 采样图像 x₀（LAION-5B，50亿对）
2. 选择时间步 t ~ U(1, 1000)
3. 添加噪声 x_t = √ᾱ_t·x₀ + √(1-ᾱ_t)·ε
4. U-Net预测 ε_pred = U-Net(x_t, t, text)
5. 计算损失 L = ||ε - ε_pred||²
6. 反向传播更新参数

**训练成本：** $600K，150K GPU小时

---

### **推理过程（4步）：**

1. 文本编码：CLIP → [77, 768]
2. 初始化噪声：x_T ~ N(0, I)
3. 逐步去噪：For t in [1000→1]（DPM++ 2M）
4. VAE解码：64×64 → 512×512

**推荐配置：**
- 步数：30
- Guidance Scale：7.5
- 采样器：DPM++ 2M

### matplotlib vs PIL 对比

**matplotlib特点：**
```
✅ 技术架构图风格
✅ 模块化设计，颜色区分
✅ 有网格线和对齐
✅ 适合技术文档

字体：通过fontsize参数控制
箭头：通过FancyArrowPatch绘制
颜色：通过facecolor/edgecolor设置
```

**PIL特点：**
```
✅ 自由绘制
✅ 完全控制每个像素
✅ 适合自定义设计

字体：通过ImageFont.truetype加载
箭头：通过draw.line和draw.polygon绘制
颜色：通过fill参数设置
```

**选择标准：**
```
技术架构图 → matplotlib
自定义插画 → PIL
照片处理 → PIL
数据可视化 → matplotlib
```
