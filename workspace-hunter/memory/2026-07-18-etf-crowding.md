## 🎯 拥挤度可以从多个维度量化

| 维度 | 逻辑 | 数据来源 | 可查性 |
|------|------|---------|--------|
| **交易拥挤** | 换手率/成交额处于历史高位，说明资金扎堆 | 日线行情（量/额） | ✅ 直接可查 |
| **融资拥挤** | 融资余额飙升 = 杠杆多头扎堆 | 融资融券数据 | ✅ Tushare/AkShare |
| **资金流向拥挤** | 北向/主力资金持续单边流入某板块 | 板块资金流 | ✅ AkShare |
| **基金持仓拥挤** | 公募基金对某行业超配到达极端 | 基金季报 | ✅ Tushare（季频） |
| **估值拥挤** | PE/PB处于历史极高分位 | 指数估值数据 | ✅ 可计算 |
| **ETF份额拥挤** | ETF规模/份额快速增长 | ETF申赎数据 | ✅ 可查 |
| **集中度拥挤** | 头部权重股市值占比过高（Herfindahl指数） | 指数成分股权重 | ✅ 可计算 |

## 💡 实操方案

我可以帮你搭一个**综合拥挤度评分模型**，输入一个行业/指数，输出各维度的历史分位 + 综合打分。大致思路：

```
拥挤度综合分 = w1×换手率分位 + w2×融资余额分位 + w3×估值分位 
             + w4×资金流入分位 + w5×基金超配分位
```

每个维度都算当前值在过去1-3年的**百分位排名**（0-100%），越接近100%越拥挤。
