## 🔥 Stable Diffusion PPT 制作经验（2026-03-29）

### 核心原则（必须遵守）

**1. 自我检查铁律** ⭐⭐⭐⭐⭐
```
生成PPT后，必须自我检查：
✅ 版面布局是否合理
✅ 字体大小是否合适
✅ 文字是否超出边框
✅ 图片是否清晰可见
✅ 图文是否紧密结合

❌ 绝对不能偷懒跳过检查步骤！
```

**2. 动态布局原则** ⭐⭐⭐⭐⭐

**根据内容量智能调整字体大小：**

| 内容量 | 字体大小 | 布局策略 | 应用场景 |
|--------|----------|----------|----------|
| ≤8行 | **16pt大字体** | 宽松舒适，突出重点 | 目录、参数说明 |
| 9-15行 | **12pt中等字体** | 平衡美观与信息量 | 基础介绍、核心总结 |
| 16-20行 | **11pt较小字体** | 稍紧凑，信息丰富 | 组件详解 |
| ≥21行 | **10pt小字体** | 紧凑排列，信息密集 | 整体架构、训练过程 |

**3. 字体统一规范** ⭐⭐⭐⭐

```
中文字体：微软雅黑
英文字体：Arial / Calibri
标题字体：微软雅黑 Bold
代码字体：Consolas / Source Code Pro
```

**4. 图文并茂标准** ⭐⭐⭐⭐⭐

**满分定义（100分）：**

1. **图片占60-70%** - 8.5×5.6-6.25英寸（大图清晰，远处可见）
2. **文字占30-40%** - 3.8英寸宽（详细说明）
3. **编号系统** - 1️⃣2️⃣3️⃣4️⃣对应图片组件
4. **文字解释图片** - 每个组件4个维度（输入/输出/参数/作用）
5. **紧密结合** - 图文一体，不是分离

**错误示例：**
```
❌ "这是CLIP" - 太简单
❌ 图片和文字分离 - 没有对应关系
❌ 没有编号系统 - 不知道对应哪个部分
```

**正确示例：**
```
✅
1️⃣ CLIP文本编码器
• 输入：文本描述
• 输出：77×768维向量
• 参数：123M
• 作用：理解文字含义

图片上的1️⃣指向CLIP框框，文字说明详细解释
```

**5. 边距控制铁律** ⭐⭐⭐⭐⭐

**极端边距控制：**

| 元素 | 尺寸 | 说明 |
|------|------|------|
| 图片 | 8.5×5.6-6.25英寸 | 根据内容量动态调整 |
| 文字框 | 3.8×5.4-5.8英寸 | 确保不超框 |
| 字体 | 10-16pt | 根据内容量动态调整 |
| 行间距 | 1.0-1.2 | 紧凑，节省空间 |
| 段后间距 | 2-3pt | 最小化 |
| 底部边距 | ≥0.8英寸 | 安全区 |

**失败历史（必须避免）：**
```
v1：5.5英寸高度 → 超框 ❌
v2：5.2英寸高度 + 12pt字体 → 仍超框 ❌
v3：5.4英寸高度 + 10pt字体 → 预计不超框 ✅
v4：10pt字体统一 → 一刀切，不合理 ❌
v5：动态布局 → 根据内容量调整 ✅
```

**6. 内容表达原则** ⭐⭐⭐⭐⭐

**必须用完整、形象、通俗易懂的句子：**

❌ **错误（短句）：**
```
Skip Connections传递细节
CLIP文本编码器
VAE压缩图像
```

✅ **正确（完整句子）：**
```
Skip Connections（跳跃连接）就像给编解码器搭建了一座桥梁，让编码器阶段的重要细节信息能够直接传递给对应的解码器层，避免信息在压缩过程中丢失。比如64×64层的细节会直接传给64×64层。

CLIP在4亿张图片+文字对上训练，能理解文字和图像的关系，把你的话翻译成AI能理解的"数字语言"。

VAE把512×512×3=786,432个数值压缩成64×64×4=16,384个数值，压缩48倍，但肉眼几乎看不出差别，就像ZIP压缩软件。
```

**7. 技术工具选择** ⭐⭐⭐⭐

**python-pptx库：**
- ✅ 完全控制布局
- ✅ 动态调整字体
- ✅ 自动化生成
- ❌ 不支持动画（需手动添加）

**使用场景：**
- 需要精确控制布局 → python-pptx
- 需要快速生成 → 2slides MCP（付费）
- 需要动画效果 → 手动添加或Google Slides API

### PPT 制作工作流程（2026-04-09 更新：5+1模式）

**⚠️ 优先使用多Agent模式！详见 `memory/multi-agent-ppt-workflow.md`**
**⚠️ 制作PPT前必须先读 `~/.openclaw/workspace-weaver/skills/powerpoint-pptx/SKILL.md`！**

**标准流程（5+1模式）：**
```
Step 1: 调研子代理（Sub-Agent A）
   ├─ 收集专业知识，写入 memory/{topic}-research.md
   └─ 先读已有经验再补充，不是从零开始

Step 2: 内容逻辑设计子代理（Sub-Agent B）⭐ 新增
   ├─ 设计每页文案、比喻、叙事逻辑、前后衔接
   ├─ 输出 memory/{topic}-ppt-content-logic.md
   └─ 与 Step 3 可并行执行

Step 3: 架构图设计子代理（Sub-Agent C，可选）
   ├─ 用 python-pptx 生成架构图/流程图
   └─ 输出独立的 diagrams.pptx 文件

Step 4: 主代理制作（Main Agent）
   ├─ 整合调研+逻辑设计+架构图，生成完整PPT
   ├─ 所有创意决策由主代理做
   └─ 必须严格遵循 Step 2 的逻辑设计文档

Step 5: 质检子代理（Sub-Agent D）
   ├─ 7维度评分（满分100）
   └─ <95分则进入修复循环

Step 5.5: 修复循环（Main Agent）
   ├─ 根据质检报告修复问题
   ├─ 重新质检直到 ≥95
   └─ 交付给用户
```

**关键原则：**
- 主代理保留所有创意决策权（子代理不做决策）
- 文件系统是唯一的跨代理通信渠道
- Task参数有长度限制，详细指令写到文件
- 子代理生成的代码/坐标必须验证（经常出错！）
- Step 2（逻辑设计）和 Step 3（架构图）可以并行执行

**实战数据（SD PPT项目）：**
- 4个子代理（调研+设计+质检×2）
- 2轮质检修复（52→87→95+）
- 总耗时约2小时，15页含4页架构图

**实战数据（Transformer PPT项目，进行中）：**
- 5个子代理（调研+逻辑设计+架构图+质检×2）
- 待验证

### 血泪教训（必须记住）

**1. 文字超出边框（反复出现N次）**
```
教训：
- 不能有任何侥幸心理
- 必须用极端参数（10pt字体 + 1.0行间距）
- 内容多的页面必须用小字体
- 生成后必须100%检查
```

**2. 图文分离（用户极度不满）**
```
教训：
- 图片和文字必须紧密结合
- 必须有编号系统（1️⃣2️⃣3️⃣4️⃣）
- 文字必须详细解释图片每个部分
- 不能只是"这是CLIP"这种简单描述
```

**3. 字体一刀切（不专业）**
```
教训：
- 不能所有页面都用相同字体大小
- 必须根据内容量动态调整
- 内容少用大字（16pt），内容多用小字（10pt）
- 这是专业设计的基本原则
```

**4. 没有自我检查（偷懒）**
```
教训：
- 生成后必须自我检查
- 不能直接发给用户
- 检查：版面、字体、边框、图片、图文结合
- 这是编织者的基本职责
```

**5. 描述太简单（用户反复强调）**
```
教训：
- 必须用完整句子
- 必须用形象比喻
- 必须有具体数字
- 必须让非AI专业学生能看懂
```

### 设计原则（编织者必须掌握）

**1. 内容驱动布局**
- 内容量决定字体大小
- 不是所有页面都用相同参数

**2. 视觉层次**
- 重要信息用大字突出
- 次要信息用小字
- 形成清晰的视觉层次

**3. 阅读舒适度**
- 内容少不拥挤
- 内容多不超框
- 保持舒适的阅读体验

**4. 动态调整**
- 根据实际情况灵活调整
- 不是机械套用模板
- 每页都是独立设计

**5. 自我检查**
- 生成后必须检查
- 确保质量达标
- 这是专业工作的基本要求

## ⚠️ PPT制作教训（2026-03-29）

### 严重错误清单（必须避免）

**错误1：同一主题拆成多页**
- ❌ 错误：VAE介绍拆成2页，CLIP拆成2页，U-Net拆成2页
- ✅ 正确：**同一主题必须在一页**，图文结合表达
- 原因：打断理解流程，信息分散

**错误2：PPT内容不完整**
- ❌ 错误：目录有10章，但实际只有10页，大量内容缺失
- ✅ 正确：**内容必须与目录完全匹配**，每一章都要有对应页面
- 原因：欺骗用户，降低信任

**错误3：文字反复超出画框**
- ❌ 错误：多次承诺修复，但每次都还有文字超出
- ✅ 正确：**生成后必须100%检查**，确保所有文字都在画框内
- 方法：
  * 文字框宽度 ≤ 11.5英寸（安全边界）
  * 预留边距：左右各1英寸，底部0.8英寸
  * 生成后打开PPT逐页检查

**错误4：排版大量空白**
- ❌ 错误：图片太小（5.8英寸宽），大量空白区域
- ✅ 正确：**图片和文字要刚好占满整页**
- 方法：
  * 图片宽度：8-10英寸（大而清晰）
  * 文字区域：3-4英寸宽（紧凑）
  * 左右布局：图片60-70%，文字30-40%
