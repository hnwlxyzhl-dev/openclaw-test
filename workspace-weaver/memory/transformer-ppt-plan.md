## 15页大纲

### 第1页：封面
- 标题：Transformer 架构深度解析
- 副标题：从直觉到原理，一篇看懂AI的核心引擎

### 第2页：目录
- 13个章节

### 第3页：Transformer 之前的世界
- RNN/LSTM 的三大痛点（顺序处理、长距离依赖、信息瓶颈）
- 比喻："像读书时只能一个字一个字看，看完了第一个字才能看第二个字"

### 第4页：Transformer 的核心直觉
- 一句话理解："每个词都可以与句子中的所有其他词直接交流"
- 例句："The cat sat on the mat, and it was happy"
- 用连线图展示 "it" 如何关注 "cat"
- 比喻："就像开会时，每个人可以同时和所有人交流"

### 第5页：Self-Attention 自注意力（上）— Q、K、V 是什么？
- Query（查询）："我在找什么"
- Key（键）："我有什么特征"
- Value（值）："我的实际内容"
- 图书馆比喻：搜索关键词 vs 书的标签 vs 书的内容
- 用表格或图展示 Q、K、V 的生成过程

### 第6页：Self-Attention 自注意力（下）— 计算过程
- 5步计算流程图：
  1. 输入 → 生成 Q、K、V
  2. Q × K^T → 计算注意力分数
  3. ÷ √d_k → 缩放（为什么？防止分数太大）
  4. Softmax → 转成概率
  5. × V → 加权求和
- 每步用通俗语言解释

### 第7页：Multi-Head Attention 多头注意力
- 为什么需要多头？比喻："一个专家只能看到一面，多个专家能从不同角度分析"
- 8个头 = 8个不同的"视角"
- 架构图：输入 → 拆分8份 → 8个独立注意力 → 合并 → 输出
- 强调：计算量和单头一样多！

### 第8页：Cross-Attention 交叉注意力
- 与 Self-Attention 的区别：
  - Self-Attention：Q、K、V 来自同一个序列（自己和自己交流）
  - Cross-Attention：Q 来自一个序列，K、V 来自另一个序列（向另一个序列提问）
- 应用场景：翻译（英文→中文）、图像描述（图片→文字）
- 架构图对比

### 第9页：Position Encoding 位置编码
- 为什么需要？Transformer 不知道词的顺序！
- 比喻："如果你把一句话的词打乱，Transformer 觉得意思一样"
- 三种主要方法对比（表格）：
  1. 正弦余弦（原始论文，用数学公式算位置）
  2. 可学习（BERT/GPT，让模型自己学）
  3. RoPE（现代模型用，旋转矩阵注入位置）

### 第10页：Transformer 整体架构
- 大架构图：Encoder-Decoder
- 左边：编码器（6层，处理输入）
- 右边：解码器（6层，生成输出）
- 数据流向箭头
- 各组件标注清晰

### 第11页：Encoder 编码器详解
- 内部结构图：
  - 输入 → 多头自注意力 → 残差+归一化 → 前馈网络 → 残差+归一化 → 输出
- 残差连接比喻："高速公路的匝道，信息可以抄近道"
- 层归一化比喻："给数据做标准化体检"

### 第12页：Decoder 解码器详解
- 内部结构图（三个子层）：
  - 掩码自注意力 → 交叉注意力 → 前馈网络
- 掩码机制比喻："考试时不能偷看后面的题"
- 交叉注意力比喻："翻译时回头查看原文"

### 第13页：训练与推理
- 训练：Teacher Forcing（把正确答案直接告诉模型）
- 推理：一步一步生成（每步只看前面的词）
- 对比图：训练 vs 推理
- 比喻："训练就像带答案做练习题，推理就像闭卷考试"

### 第14页：Transformer 的演进与应用
- 三大家族：
  - Encoder-only（BERT）：理解类任务（搜索、分类）
  - Decoder-only（GPT）：生成类任务（对话、写作）
  - Encoder-Decoder（T5）：翻译、摘要
- 代表模型列表
- 应用领域：NLP、CV、语音、多模态

### 第15页：总结
- 5个核心要点回顾
- Transformer 的本质："让每个信息单元都能与所有其他信息单元直接交流"
- 学习资源推荐
