## 1. 为什么需要 Transformer

### 1.1 背景：2017 年之前的 AI 语言处理方式

在 2017 年 Transformer 诞生之前，人工智能处理人类语言主要依赖一种叫做**循环神经网络**（Recurrent Neural Network，简称 RNN）的技术。RNN 的工作原理很简单：它把文本当作一个"序列"，一个词一个词地依次读进去，每读一个词就更新一次内部的"记忆状态"，读完所有词之后，根据积累的记忆来做判断（比如翻译、分类、生成等）。

后来人们又发明了 **LSTM**（Long Short-Term Memory，长短期记忆网络），这是 RNN 的改进版本。LSTM 引入了"门控机制"（gating mechanism），可以更聪明地决定哪些信息该记住、哪些该忘记，在一定程度上缓解了 RNN 的记忆衰减问题。但 LSTM 的核心思想仍然和 RNN 一样——**逐个处理、顺序传递**。

从 1990 年代到 2017 年，RNN/LSTM 家族统治了自然语言处理（Natural Language Processing，简称 NLP）领域将近 20 年。几乎所有机器翻译、语音识别、文本生成的系统，底层都是 RNN 或 LSTM。

### 1.2 RNN/LSTM 的三大问题

尽管 RNN/LSTM 取得了不少成就，但它们存在三个根本性的缺陷，这些缺陷最终限制了 AI 处理语言的能力天花板。

#### 问题一：顺序处理——像读书只能一个字一个字地看

RNN 必须一个词一个词地处理文本，读完第一个词之后才能处理第二个词，处理完第二个词之后才能处理第三个——这是一个严格的**串行过程**（serial process）。这就好比你看一本书，必须先看懂第一个字，才能看第二个字，完全无法同时理解整句话的意思。

**串行处理带来的致命后果是无法利用并行计算。** 现代图形处理器（GPU）拥有数千个计算核心，可以同时执行大量运算。但 RNN 的串行特性意味着它只能用一个核心工作，其余数千个核心全部闲置。这就好比你雇了 1000 个工人来搬砖，但规定每次只能有 1 个人搬，其余 999 个人在旁边看着——资源浪费极为严重。

**具体数字对比：** 在原始 Transformer 论文中，Transformer 在 8 块 NVIDIA P100 GPU 上仅需训练 12 小时就达到了当时最优的翻译效果（WMT 2014 英德翻译 28.4 BLEU 分）。而之前基于 LSTM 的最强模型需要在更多 GPU 上训练数天才能达到类似的水平。训练效率的差异不是 2 倍、3 倍，而是**数十倍**。

#### 问题二：长距离遗忘——读到结尾就忘了开头

当句子或文档很长时，RNN 在处理后面的词时会逐渐"忘记"前面的信息。这种现象在技术上叫做**梯度消失问题**（vanishing gradient problem）——当你通过"反向传播"（backpropagation，一种训练技术）来更新模型参数时，误差信号在经过几十上百步的传递后会逐渐衰减到接近零，导致模型无法有效学习到长距离的依赖关系。

**用更直观的话说：** 想象你在听一个长达 10 分钟的故事，听到最后时你已经记不清开头发生了什么。RNN 的"记忆"也是这样——信息每经过一步处理都会被压缩和衰减，经过 50 步之后，最初的信息几乎已经丢失殆尽。

**实验数据：** 研究表明，当句子长度超过约 50 个词时，标准 RNN 的性能就会明显下降。即使是最先进的 LSTM，在处理超过 200-300 个词的长文本时，也会出现明显的"遗忘"现象。但在实际应用中，我们经常需要处理几千甚至几万词的长文档——这对 RNN/LSTM 来说几乎是不可能完成的任务。

#### 问题三：信息瓶颈——所有内容被压缩成一根针眼

RNN 在处理完整个输入序列之后，需要把所有信息压缩成一个**固定长度的向量**（fixed-length vector），然后再基于这个向量进行翻译或回答。这就好比你读了一本 500 页的书，却只能用一个 100 字的摘要来做所有判断——无论你怎么压缩，信息损失都是巨大的。

**具体来说：** 假设输入是一段 100 个词的英文，RNN 需要把这 100 个词的信息全部压缩到一个比如 512 维的向量里（512 个浮点数），然后用这个向量来生成中文翻译。512 个浮点数能表达多少信息？远远不够。就像你试图用一个便签纸记录一整场 2 小时会议的所有内容——你只能记下最关键的几个点，大量细节都会丢失。

2014 年，Bahdanau 等人引入了**注意力机制**（attention mechanism），允许模型在翻译时"回头看"输入序列的不同部分，部分缓解了信息瓶颈问题。但注意力机制只是附加在 RNN 之上的一个"补丁"，RNN 的顺序处理和记忆衰减这两个根本问题依然存在。

### 1.3 Transformer 的核心突破

2017 年，Google 的一个研究团队在论文《Attention Is All You Need》中提出了一个革命性的想法：**彻底抛弃 RNN 和 CNN，仅用注意力机制来建模序列关系。** 这就是 Transformer。

Transformer 的核心思想可以用一句话概括：**让序列中的每个词都能与所有其他词直接交流。** 如果把 RNN 比作"排队发言"（每个人必须等前面的人说完才能说话），那 Transformer 就像是"圆桌会议"——每个人都可以同时听到所有人说的话，并且可以根据自己的需要关注任何一个人。

这个看似简单的想法，同时解决了 RNN 的三个问题：

| RNN 的问题 | Transformer 的解决方案 |
|-----------|----------------------|
| 顺序处理，无法并行 | 每个词同时计算注意力，完全并行化 |
| 长距离遗忘 | 每个词直接与所有词交流，不存在信息衰减 |
| 信息瓶颈 | 不需要压缩成固定向量，每层都保留完整信息 |

**关于命名：** Transformer 这个名字可能让人联想到变形金刚，但实际上它取自英文单词"transform"（变换），意思是这个架构可以把一种序列"变换"成另一种序列。当然，这个巧合的名字也恰好体现了它的强大——它确实"变形"了整个 AI 领域。

## 2. Transformer 架构概述

### 2.1 整体结构：左边理解，右边生成

原始 Transformer 采用**编码器-解码器架构**（Encoder-Decoder architecture）。这个架构由两大模块组成：

- **编码器（Encoder）**：负责"理解"输入内容。它接收输入序列（比如一个英文句子），经过多层处理后输出一组包含丰富语义信息的向量。你可以把编码器想象成一个"阅读理解专家"——它读入一篇文章，提炼出文章的核心含义。
- **解码器（Decoder）**：负责"生成"输出内容。它接收编码器的输出作为参考，逐词生成目标序列（比如对应的中文翻译）。你可以把解码器想象成一个"翻译专家"——它根据编码器提供的理解，一个词一个词地写出翻译。

两个模块之间通过**交叉注意力**（Cross-Attention）连接——这是编码器和解码器之间唯一的"桥梁"，解码器通过它来"查阅"编码器对原文的理解。

**一个完整的翻译流程比喻：** 想象一家翻译公司。客户送来一篇英文文章（输入），交给编辑部（编码器），编辑部的编辑们仔细阅读文章，做详细的笔记和标注（编码器的多层处理）。然后，编辑部把笔记交给翻译部（解码器），翻译部的翻译官一边参考笔记，一边逐字逐句地写出中文翻译（解码器的自回归生成）。翻译官每写一个词，都会参考编辑部的笔记来确保准确性（交叉注意力）。

### 2.2 编码器的内部结构

编码器由 N 个相同的层堆叠而成（原始论文中 N=6）。每一层包含两个核心子模块：

1. **多头自注意力（Multi-Head Self-Attention）**：让每个位置都能关注输入序列中的所有位置。这是编码器的"信息交流中心"——所有词在这里互相交换信息。
2. **前馈网络（Feed-Forward Network，也叫 FFN）**：对每个位置独立地应用相同的全连接网络。这是编码器的"信息加工厂"——每个词在这里独立地深化自己的理解。

每个子模块后面都有一个**残差连接**（Residual Connection）加上**层归一化**（Layer Normalization）。残差连接的作用是保留原始信息，层归一化的作用是稳定数值范围。两者合在一起就像高速公路的"匝道+限速标识"——确保信息既不会在深层网络中丢失，也不会因为数值过大而失控。

**编码器单层的数据流：**
```
输入 → [多头自注意力] → 残差连接 + 层归一化 → [前馈网络] → 残差连接 + 层归一化 → 输出（传给下一层）
```

### 2.3 解码器的内部结构

解码器也由 N 个相同的层堆叠而成（原始论文中 N=6），但每一层包含**三个**子模块（比编码器多一个）：

1. **掩码多头自注意力（Masked Multi-Head Self-Attention）**：与编码器的自注意力类似，但多了一个"因果掩码"（causal mask），确保每个词只能看到它之前（包括自己）的词，不能"偷看"未来的词。这就像考试时不能翻到后面去看还没考的题目。
2. **编码器-解码器注意力（Encoder-Decoder Attention，即交叉注意力）**：解码器的 Query 来自上一层输出，Key 和 Value 来自编码器的最终输出。这是解码器"查阅"编码器的唯一通道。
3. **前馈网络（Feed-Forward Network）**：与编码器的 FFN 相同，对每个位置独立处理。

**解码器单层的数据流：**
```
输入 → [掩码自注意力] → 残差+归一化 → [交叉注意力] → 残差+归一化 → [前馈网络] → 残差+归一化 → 输出
```

### 2.4 输入和输出处理

在进入编码器和解码器之前，原始文本需要经过以下处理：

**编码器端：**
```
原始文本 "I love you" 
  → 分词（Tokenization）→ ["I", "love", "you"] 
  → 词嵌入（Token Embedding）→ 每个词变成一个 512 维的向量
  + 位置编码（Position Encoding）→ 注入位置信息
  → 送入编码器
```

**解码器端（训练时）：**
```
目标文本向右移一位：[<START>, "我", "爱", "你"]
  → 分词 + 词嵌入 + 位置编码
  → 送入解码器
  → 经过线性层 + Softmax
  → 输出每个位置上词表中所有词的概率分布
```

**解码器端（推理时）：**
```
起始符号 <START>
  → 送入解码器 → 预测第 1 个词 "我"
  → 将 "我" 加入输入 → 预测第 2 个词 "爱"
  → 将 "爱" 加入输入 → 预测第 3 个词 "你"
  → 将 "你" 加入输入 → 预测结束符 <END> → 停止
```

### 2.5 原始 Transformer 的具体配置

| 超参数 | 值 | 说明 |
|--------|-----|------|
| 编码器层数 | 6 | 编码器有 6 个相同的层 |
| 解码器层数 | 6 | 解码器有 6 个相同的层 |
| 模型维度 d_model | 512 | 每个词向量的维度 |
| 注意力头数 | 8 | 多头注意力的头数 |
| 每个头的维度 d_k | 64 | 512 / 8 = 64 |
| 前馈网络隐藏维度 d_ff | 2048 | 通常是 d_model 的 4 倍 |
| Dropout 率 | 0.1 | 防止过拟合的正则化手段 |
| 优化器 | Adam | 自适应学习率优化器 |
| 参数总量 | 约 6500 万 | 相比今天的模型非常小 |
| 训练硬件 | 8 块 NVIDIA P100 GPU | |
| 训练时间 | 12 小时（英德翻译） | |

**与现代模型的对比：** 原始 Transformer 只有 6500 万参数，而今天的 GPT-4 据传有约 1.8 万亿参数（增长了近 3 万倍），LLaMA-3 有 70B-405B 参数，但这些模型的核心架构与原始 Transformer 基本相同——只是层数更多（32-80 层）、维度更大（4096-8192 维）、数据更多而已。

## 3. 训练阶段详解

### 3.1 编码器的训练过程

#### 3.1.1 词嵌入（Token Embedding）——把文字变成数字

计算机无法直接理解文字，所以第一步需要把文字转换成数字。这个过程叫做**词嵌入**（word embedding）。

**分词（Tokenization）：** 首先，原始文本会被切成一个个最小的单元，叫做 **token**（词元）。token 可以是一个完整的词（如 "love"），也可以是一个词的一部分（如 "un-" 和 "-able"），甚至可以是一个字符。原始 Transformer 使用了一种叫做 Byte-Pair Encoding（BPE，字节对编码）的分词方法，它会统计训练数据中最常见的字符组合，逐步将高频组合合并成更大的 token。这种方法的巧妙之处在于：常见的词会被保留为完整的 token（如 "the"、"apple"），而罕见的词会被拆成更小的片段，从而避免了"词表外词"（Out-of-Vocabulary，OOV）的问题。

**嵌入矩阵（Embedding Matrix）：** 分词之后，每个 token 会被映射成一个向量。这个映射是通过查询一个**嵌入矩阵**来实现的——嵌入矩阵的每一行对应词表中的一个 token，每一行是一个 d_model 维的向量（原始 Transformer 中是 512 维）。比如词表有 30,000 个 token，嵌入矩阵的大小就是 30,000 x 512。

**比喻：嵌入矩阵就像一本超级字典。** 普通字典告诉我们每个词的意思，嵌入矩阵告诉我们每个词的"数字画像"——一个由 512 个数字组成的向量。这些数字不是随机的，而是在训练过程中自动学习出来的：意思相近的词（如 "happy" 和 "joyful"）在向量空间中会靠得很近，意思不同的词（如 "happy" 和 "sad"）会离得很远。

**具体数字：** 原始 Transformer 的词表大小为 37,000（包含子词单元），嵌入维度为 512，所以嵌入矩阵包含 37,000 x 512 约 1890 万个参数——占整个模型参数量的近 30%。

#### 3.1.2 位置编码（Position Encoding）——告诉模型词的顺序

自注意力机制有一个重要的特性：它是**排列不变的**（permutation invariant）。也就是说，如果你把一句话的词打乱顺序再输入，自注意力的计算结果不会变。对于语言来说这是不可接受的——"狗咬人"和"人咬狗"意思截然不同，但它们的词完全相同。

**比喻：** 自注意力就像一群蒙着眼睛的人在圆桌上讨论。他们能听到每个人的发言内容（词的含义），但不知道每个人坐在哪个位置（词的顺序）。位置编码就像是给每个人的椅子贴上了编号——现在每个人不仅知道别人说了什么，还知道别人坐在第几把椅子上。

原始 Transformer 使用**正弦余弦位置编码**（Sinusoidal Position Encoding），用不同频率的 sin 和 cos 函数为每个位置生成一个 512 维的向量。这种方法的巧妙之处在于它不需要额外的学习参数，而且理论上可以处理比训练时更长的序列。

**位置编码与词嵌入的合并方式：** 直接将两者相加（element-wise addition），得到最终的输入向量。这就像给每个词的"身份证"上加了一个"住址"——既保留了词本身的含义，又包含了它在句子中的位置信息。

#### 3.1.3 QKV 矩阵——为每个词准备"搜索工具"

进入编码器的每一层后，输入向量首先会经过多头自注意力模块。自注意力的第一步是为每个词生成三组向量：**Query**（查询）、**Key**（键）和 **Value**（值）。

**生成方式：** 每个词的输入向量（512 维）分别乘以三个不同的权重矩阵 W^Q、W^K、W^V（每个矩阵的大小都是 512 x 64），得到三个 64 维的向量。注意，这些权重矩阵是**可学习的参数**（learnable parameters）——它们在训练过程中会不断调整，最终学会为不同语境下的同一个词生成不同的 Q、K、V。

**比喻——图书馆搜索系统：**
- **Query（查询）**：你在图书馆搜索时输入的关键词，代表"我在找什么信息"。比如你正在处理句子中的 "it"，你的 Query 就可能编码了"我需要找到一个名词来告诉我 it 指的是谁"。
- **Key（键）**：每本书的书名和标签，代表"我有什么特征可以被别人找到"。比如 "cat" 这个词的 Key 可能编码了"名词、动物、单数、可作主语"等特征。
- **Value（值）**：书的实际内容，代表"我的真实信息是什么"。当 "it" 通过 Query 找到 "cat" 的 Key 匹配度最高后，它就会读取 "cat" 的 Value 来丰富自己的含义。

**关键点：每个词同时扮演"搜索者"和"被搜索者"的角色。** 它用自己的 Query 去搜索所有词的 Key，同时自己的 Key 也被所有其他词的 Query 搜索。

#### 3.1.4 Attention 计算的五步过程

自注意力的计算可以分为五个清晰的步骤，我们继续用图书馆搜索的比喻来解释：

**步骤 1：生成 Q、K、V** —— 输入的每个词经过三个权重矩阵变换，生成各自的 Query、Key、Value 向量。在多头注意力中，这个过程会同时进行 8 次（8 个头），每个头有自己独立的权重矩阵。这一步可以同时对所有词并行执行。

**步骤 2：计算注意力分数（Q x K^T）** —— 把每个词的 Query 向量和所有词的 Key 向量做**点积**（dot product）运算。点积的值越大，说明两个词的关联度越强。对于一个有 T 个词的句子，这一步会产生一个 T x T 的分数矩阵。比喻：就像你把搜索关键词和每本书的标签逐一比对，给每本书打一个匹配分数。

**步骤 3：缩放（除以根号 d_k）** —— 将分数除以根号 64 即 8。当向量维度较高时，点积的结果会变得非常大，导致下一步的 Softmax 函数进入"饱和区"——只有最高分接近 1，其他分数都接近 0。除以根号 d_k 就像给分数做一个"标准化校准"。数学上，如果 Q 和 K 的各分量独立且均值为 0、方差为 1，则点积的方差为 d_k，除以根号 d_k 后方差变为 1。

**步骤 4：Softmax 归一化** —— 对分数矩阵的每一行做 Softmax 运算，将分数转换成 0 到 1 之间的概率值，且每一行的和等于 1。Softmax 会把最高分放大，较低分缩小，但不会把任何一个完全变成 0。比喻：就像你根据匹配分数决定在每本书上花多少时间——匹配度最高的书花最多时间，但其他相关的书也会看一些。

**步骤 5：加权求和（x V）** —— 将 Softmax 后的概率权重与每个词的 Value 向量相乘并求和，得到每个词的最终输出。关联度越高的词，其 Value 在输出中占的比重越大。比喻：你根据在每本书上分配的时间比例提取信息，混合成最终答案。对于 "it" 来说，"cat" 的注意力权重最高，所以 "it" 的输出中包含了大量 "cat" 的语义信息。

#### 3.1.5 残差连接与层归一化

自注意力计算完成后，结果不会直接传给下一个模块，而是先经过**残差连接**和**层归一化**（Layer Normalization，简称 LayerNorm）。

**残差连接（Residual Connection）：** 公式为"输出 = x + 子模块(x)"。将子模块的输出与原始输入直接相加。比喻：残差连接就像高速公路旁边的匝道——如果主干道（注意力层）堵了或出了问题，信息可以直接从匝道通过，不受阻碍。对于深层网络来说，残差连接解决了梯度消失问题——梯度可以直接通过残差路径回传到任意一层。

**层归一化（Layer Normalization）：** 对每个词的向量做标准化处理——先计算均值和标准差，然后将每个元素减去均值、除以标准差，最后乘以一个可学习的缩放参数。比喻：层归一化就像给数据做"标准化体检"——确保每个词的向量都在相似的数值范围内，不会出现某些维度特别大、某些维度特别小的情况。

**Post-LN 与 Pre-LN：** 原始 Transformer 使用后置归一化（先加残差再归一化），现代模型普遍使用前置归一化（先归一化再做子层再加残差），后者训练更稳定。

#### 3.1.6 前馈网络 / MLP

前馈网络（Feed-Forward Network，简称 FFN）由两个线性变换和一个非线性激活函数组成。先通过一个线性层将维度从 512 扩大到 2048（扩大 4 倍），再通过 ReLU 激活函数引入非线性，最后通过另一个线性层将维度从 2048 压回 512。

**比喻：** 如果自注意力是"圆桌会议"（词与词之间交流信息），那前馈网络就是"独立思考时间"。每个词在听完所有人的发言之后，需要一个安静的环境来消化和深化自己收集到的信息。

**为什么需要"扩展再压缩"？** 先把维度从 512 扩大到 2048，相当于给每个词一个更大的"思考空间"——在这个更大的空间里，词可以做更复杂的特征提取。然后再压缩回 512，相当于把思考的结果浓缩成一个精炼的表示。

**计算量占比：** 有趣的是，前馈网络的计算量实际上比注意力层还大。在原始 Transformer 中，注意力层的参数量约为 512 x 512 x 3 x 3（Q、K、V、输出投影，乘以头数）约 236 万，而 FFN 的参数量为 512 x 2048 x 2 约 210 万。但由于 FFN 的隐藏维度是模型维度的 4 倍，FFN 的计算量约为注意力的 2 倍。

### 3.2 解码器的训练过程

#### 3.2.1 解码器的输入和输出

在训练阶段，解码器的处理方式与推理阶段有本质区别。训练时，解码器一次性接收**完整的目标序列**（向右移一位），可以并行计算所有位置的输出。

**具体例子：** 假设要把 "I love you" 翻译成 "我爱你"。

训练时解码器的输入是：[START, 我, 爱]
训练时解码器应该输出的目标（标签）是：[我, 爱, 你, END]

注意输入和目标之间"错开一位"的关系——模型看到 [START, 我, 爱] 后，应该预测出下一个词是 [我, 爱, 你]。这种"看到前面的词，预测下一个词"的模式就是语言建模的核心。

**比喻：训练就像开卷考试。** 考卷上的每一道题（每个位置），学生都可以看到前面所有题的正确答案（之前的词），只需要预测下一道题的答案。这让学生可以高效地学习——每做错一道题，立刻就知道正确答案是什么，可以快速纠正。

#### 3.2.2 掩码自注意力（Masked Self-Attention）

解码器的第一层是掩码多头自注意力。它的计算过程与编码器的自注意力完全相同（同样有 Q、K、V 五步计算），但多了一个关键操作：**因果掩码**（causal mask）。

**因果掩码的工作原理：** 在注意力分数矩阵中，将当前位置之后的所有位置的分数设为负无穷大。经过 Softmax 之后，负无穷大的位置会变成 0——也就是说，这些位置的注意力权重为 0，等于"看不见"。

**具体例子：** 假设输入是 [START, 我, 爱]。当处理"爱"这个词时，因果掩码会遮住"我"之后的所有位置（虽然在这个例子中没有更后面的词）。但如果有更多词，比如处理第 5 个词时，因果掩码会确保它只能看到第 1-5 个词，不能看到第 6 个及之后的词。

**比喻：** 因果掩码就像考试时在试卷上画了一条竖线——你只能看到竖线左边（已经考过的）的题目和答案，竖线右边（还没考的）的题目被遮住了。这确保了模型在预测下一个词时，不会"偷看"答案。

**为什么训练时也需要掩码？** 虽然训练时我们有完整的目标序列，但如果让模型在训练时也能看到未来的词，那么模型学到的是一种"作弊"的策略——它会直接从后面的词"抄答案"，而不是真正学会从上下文预测下一个词。到了推理时（没有正确答案可以抄），模型的表现就会一塌糊涂。因果掩码确保训练和推理的行为一致。

#### 3.2.3 交叉注意力（Cross-Attention）

解码器的第二层是交叉注意力。这是编码器和解码器之间唯一的连接通道。

**计算方式：** 交叉注意力的 Q 来自解码器上一层（掩码自注意力）的输出，K 和 V 来自编码器的最终输出。计算过程与自注意力完全相同：Q x K^T -> 缩放 -> Softmax -> x V。

**比喻——翻译官查阅编辑笔记：** 如果说自注意力是"自己人内部开会讨论"，那交叉注意力就是"带着问题去查阅参考资料"。解码器在生成每个词时，都会通过交叉注意力去"查阅"编码器对原文的理解。当解码器要生成"爱"这个词时，它的 Query（"我在找一个表示感情的动词"）会与编码器中每个英文词的 Key 做匹配，发现 "love" 的匹配度最高，于是提取 "love" 的 Value 来辅助生成"爱"。

**关键区别：** 自注意力的 Q、K、V 全部来自同一个序列（同一个"团队"内部交流），而交叉注意力的 Q 来自一个序列（解码器），K 和 V 来自另一个序列（编码器）。就像一个是"团队内部讨论"，一个是"跨部门咨询"。

#### 3.2.4 输出层：线性变换 + Softmax

解码器的最后一层输出经过一个线性变换层和 Softmax 函数，转换为词表上所有词的概率分布。

**线性变换：** 将每个位置的输出向量（512 维）映射到词表大小的维度（37,000 维）。这个映射是通过一个权重矩阵（512 x 37,000）实现的。有意思的是，在很多实现中，这个权重矩阵与嵌入矩阵是**共享的**（weight tying）——也就是说，输出层的权重就是嵌入矩阵的转置。这减少了参数量，也让"编码一个词"和"解码一个词"共享了同一个语义空间。

**Softmax：** 将 37,000 维的原始分数转换成概率分布——每个词有一个 0 到 1 之间的概率值，所有概率之和为 1。概率最高的词就是模型"认为"最应该出现在这个位置的词。

### 3.3 损失函数和训练过程

#### 3.3.1 交叉熵损失（Cross-Entropy Loss）

**交叉熵是什么？** 交叉熵是信息论中的一个概念，用来衡量两个概率分布之间的"距离"。在机器学习中，交叉熵损失函数衡量的是模型预测的概率分布与真实标签（one-hot 分布）之间的差距。

**通俗解释：** 假设你在做选择题，每题有 4 个选项（A、B、C、D）。正确答案是 B。如果你对每个选项的信心程度（概率）是 A:0.1, B:0.7, C:0.15, D:0.05，那交叉熵就是 -log(0.7) 约等于 0.36。如果你对正确答案更自信，比如 B:0.95，交叉熵就是 -log(0.95) 约等于 0.05——更小的交叉熵意味着更好的预测。

**公式：** 对于一个位置，如果正确答案是词 w，模型预测 w 的概率为 p(w)，则该位置的损失为：
```
Loss = -log(p(w))
```
如果模型对正确词的预测概率很高（接近 1），损失接近 0；如果预测概率很低（接近 0），损失趋向无穷大。

**整个序列的损失：** 将所有位置的损失取平均值。对于一个 10 个词的目标序列，总损失就是 10 个位置的交叉熵的平均值。

**具体数字：** 在训练初期，模型的预测几乎是随机的（词表有 37,000 个词，每个词的概率约 1/37000），此时交叉熵约为 -log(1/37000) 约等于 10.5。经过充分训练后，模型对正确词的预测概率通常能达到 0.5-0.9，交叉熵降低到 1-3 左右。

#### 3.3.2 Teacher Forcing（教师强制）

**什么是 Teacher Forcing？** 在训练时，解码器的输入不是模型自己之前预测的词，而是**真实的目标序列**（正确答案）。也就是说，每一步解码器看到的都是"标准答案"，而不是自己可能出错的前面几步。

**具体例子：** 假设目标序列是"我爱你"。
- **不使用 Teacher Forcing：** 解码器预测第 1 个词（比如预测成"他"，错了），然后用"他"作为输入去预测第 2 个词——错误会不断累积和放大。
- **使用 Teacher Forcing：** 解码器在预测第 1 个词时看到 [START]，预测第 2 个词时看到 [START, 我]（真实的"我"，不是模型预测的），预测第 3 个词时看到 [START, 我, 爱]（真实的"爱"）——每一步都基于正确的前文来学习。

**比喻：** 不使用 Teacher Forcing 就像"闭卷考试，而且前面答错了不能改"——一个错误会导致后面全部跟着错。使用 Teacher Forcing 就像"开卷考试，每做完一道题老师立刻告诉你正确答案"——你可以基于正确答案继续学习下一题，效率高得多。

**Teacher Forcing 的优缺点：**

优点：
- 训练效率极高——模型不需要等到自己生成完整个序列才能学习，每一步都可以立即获得正确的反馈。
- 梯度信号更稳定——因为每一步的输入都是正确的，不会出现"错误累积"导致的梯度不稳定问题。
- 训练速度更快——可以并行计算所有位置的损失。

缺点：
- **暴露偏差（Exposure Bias）：** 训练时模型看到的都是正确的输入，但推理时看到的是自己之前生成的（可能有错的）输出。这种"训练和推理的输入分布不一致"可能导致模型在推理时表现不佳——就像一个学生平时做练习题都有答案参考，到了真正考试时突然没有答案了，就不知所措了。

**缓解暴露偏差的方法：** 常见的方法包括 Scheduled Sampling（以一定概率随机使用模型自己的预测作为输入，而不是总用正确答案）和在训练时故意引入一些噪声。

#### 3.3.3 完整的训练过程

**训练数据准备：** 原始 Transformer 在 WMT 2014 英德翻译数据集上训练，包含约 450 万个句对。每个句对包括一个英文句子和一个德语句子。

**训练步骤：**

1. **编码器处理输入：** 将英文句子经过分词、嵌入、位置编码后送入编码器，经过 6 层处理，得到编码输出。这一步可以并行处理所有词。

2. **解码器处理目标：** 将德语目标序列（向右移一位）经过分词、嵌入、位置编码后送入解码器。在解码器的每一层中，先做掩码自注意力（只能看到之前的词），再做交叉注意力（查阅编码器的输出），最后做前馈网络。

3. **计算损失：** 解码器的输出经过线性层和 Softmax 后，得到每个位置上所有词的概率分布。将预测概率与真实标签对比，计算交叉熵损失。

4. **反向传播（Backpropagation）：** 根据损失函数计算每个参数的梯度（即每个参数对损失的贡献有多大），然后用优化器（Adam）更新参数。更新方向是"减小损失"的方向。

5. **重复以上步骤：** 在整个训练集上反复训练，直到损失不再明显下降。

**关键超参数：**
- 批量大小（Batch Size）：原始论文使用了约 25,000 个 token 的批量
- 学习率（Learning Rate）：峰值 0.0003，先线性 warmup 4000 步，然后按步数的 -0.5 次方衰减
- 标签平滑（Label Smoothing）：0.1，防止模型过度自信
- Dropout：0.1，随机"关闭"一些神经元，防止过拟合

**训练规模：** 总共训练了 300,000 步（约 40 个 epoch），在 8 块 P100 GPU 上耗时 12 小时。

## 4. 推理阶段详解

### 4.1 推理与训练的核心区别

在讲推理的细节之前，我们需要先理解训练和推理之间的关键差异。这些差异可以用"开卷考试 vs 闭卷考试"的比喻来贯穿。

**训练阶段（开卷考试）：**
- 解码器的输入是**完整的正确目标序列**（Teacher Forcing）
- 所有位置的损失可以**同时并行计算**
- 编码器只需处理输入**一次**
- 编码器的输出可以被所有训练样本复用（因为同一个 batch 中输入可能不同，但编码器的处理方式是相同的）

**推理阶段（闭卷考试）：**
- 解码器的输入是**模型自己之前生成的词**（没有正确答案可参考）
- 必须**逐词生成**（串行过程），每生成一个词才能预测下一个
- 编码器同样只需处理输入一次
- 编码器的输出在整个生成过程中保持不变（被缓存起来反复使用）

**比喻对比：**
- 训练就像"做练习册"——每做一道题，翻到后面看答案，然后基于正确答案继续做下一题。你可以同时看所有题的答案（并行），效率很高。
- 推理就像"真正的考试"——没有答案可看，你必须自己做每一道题，而且做完一道才能看下一道（串行），效率较低。更关键的是，如果你第一道题做错了，后面可能全部跟着错，而且你不能回头修改。

### 4.2 编码器在推理时的行为

**编码器在推理时的处理方式与训练时完全相同。** 输入序列经过分词、嵌入、位置编码后，通过编码器的 6 层处理，输出一组上下文化的向量。这个过程只需要执行一次，然后编码器的输出会被缓存（cache）起来，在整个生成过程中反复使用。

**比喻：** 编码器就像"编辑部"——客户送来一篇文章，编辑部只需要阅读和分析一次，然后就可以把分析结果（笔记）交给翻译部。翻译部在翻译整篇文章的过程中，随时可以翻阅这份笔记。

**关键优化：KV Cache。** 在现代实现中，编码器的 Key 和 Value 向量会被缓存起来，这样解码器的交叉注意力在每一步生成时就不需要重新计算编码器的 K 和 V，只需要计算解码器自己的 Q，然后与缓存的编码器 K、V 做注意力计算。

### 4.3 解码器在推理时的行为——自回归生成

**自回归生成（Autoregressive Generation）** 是推理阶段的核心机制。它的意思是：每生成一个新词，都要把这个词加入之前的序列，作为下一步的输入，然后才能预测下一个词。这个过程是串行的，无法并行化。

**具体步骤（以翻译 "I love you" 为例）：**

**步骤 1：初始化**
- 编码器处理 "I love you"，输出编码向量（只做一次，然后缓存）
- 解码器的输入是 [START]（一个特殊的起始符号）
- 解码器经过 6 层处理，输出一个向量
- 经过线性层 + Softmax，得到词表上所有词的概率分布
- 选择概率最高的词（比如"我"）作为第一个输出

**步骤 2：生成第二个词**
- 将"我"加入解码器的输入：[START, 我]
- 解码器重新处理整个输入序列
- 输出概率分布，选择概率最高的词（比如"爱"）

**步骤 3：生成第三个词**
- 将"爱"加入输入：[START, 我, 爱]
- 解码器重新处理
- 选择概率最高的词（比如"你"）

**步骤 4：检测结束**
- 将"你"加入输入：[START, 我, 爱, 你]
- 解码器输出概率最高的词是 END（结束符号）
- 生成过程结束
- 最终输出："我爱你"

**比喻——逐字逐句地写文章：** 自回归生成就像你写一篇文章——你必须先想好第一个字，写下来，再想第二个字，再写下来。你没办法同时写一整段话。而且，一旦你写了一个字，就不能回头修改（虽然有些技术如"beam search"可以在一定程度上缓解这个问题）。

### 4.4 推理效率的关键瓶颈

**为什么推理比训练慢？** 主要有以下几个原因：

**瓶颈 1：串行生成。** 训练时，解码器可以并行处理所有位置（因为所有位置的正确答案都已知）。但推理时，必须等第 1 个词生成后才能生成第 2 个词，等第 2 个词生成后才能生成第 3 个词——这是一个严格的串行过程。对于 GPU 这种擅长并行计算的硬件来说，串行计算是极大的浪费。

**瓶颈 2：重复计算。** 在最朴素（未优化）的实现中，每生成一个新词，解码器都需要重新处理之前的所有词。比如生成第 10 个词时，解码器需要重新计算前 9 个词的自注意力——但前 9 个词的注意力结果在生成第 9 个词时已经计算过了！

**KV Cache 优化：** 为了解决瓶颈 2，现代实现普遍使用 KV Cache。核心思想是：在自回归生成的每一步，只计算**新词**的 Q、K、V，然后将新词的 K、V 追加到缓存中。这样，下一步的注意力计算只需要用新词的 Q 与所有缓存的 K、V 做计算，而不需要重新计算历史词的 K、V。

**KV Cache 的大小：** 假设模型有 L 层、H 个注意力头、每个头的维度为 d_k、序列长度为 T、批量大小为 B，则 KV Cache 的大小为：
```
2 x L x T x B x H x d_k x 精度字节数
```
例如，LLaMA-2 70B（80 层、64 个头、128 维、FP16），处理 4096 个 token 时，KV Cache 大小约为 5.4 GB。

**比喻：** KV Cache 就像是"会议纪要"。每次开会（每生成一个新词），你不需要把之前所有的会议内容重新讨论一遍——你只需要查看之前保存的会议纪要（缓存的 K、V），然后只讨论新增的内容。

### 4.5 训练 vs 推理的全面对比

| 维度 | 训练 | 推理 |
|------|------|------|
| 解码器输入 | 完整的正确目标序列（Teacher Forcing） | 模型自己之前生成的词 |
| 计算方式 | 所有位置并行计算 | 逐词串行生成 |
| 掩码 | 需要因果掩码（防止看到未来词） | 同样需要因果掩码 |
| 编码器 | 处理一次，输出用于计算损失 | 处理一次，输出被缓存复用 |
| 损失函数 | 交叉熵损失（每个位置都计算） | 无损失函数，只做预测 |
| 梯度计算 | 需要反向传播更新参数 | 不需要梯度（参数固定） |
| KV Cache | 不需要（因为可以并行计算） | 必须使用（避免重复计算） |
| 效率 | 高（并行） | 低（串行） |
| 批量大小 | 可以很大（几百到几千） | 受限于显存和延迟要求 |
| 硬件利用率 | GPU 利用率高 | GPU 利用率低（串行瓶颈） |

**比喻总结：**
- **训练** = 新员工培训期。师傅手把手教，每做一步都告诉你正确答案（Teacher Forcing），错了立刻纠正。可以同时学习多个案例（大批量），效率高但需要投入大量资源。
- **推理** = 正式上岗工作。没有人告诉你答案，必须完全靠自己。每做一步都要参考之前的结果（自回归），做错了只能将错就错。但只需要一台电脑就能工作（推理资源需求低）。

### 4.6 词选择策略：从贪心到采样

推理时，每一步 Softmax 都会输出一个概率分布（词表上所有词的概率）。问题是：我们如何从这个概率分布中选择最终的词？这里有几种不同的策略。

#### 4.6.1 贪心搜索（Greedy Search）

**策略：每一步都选择概率最高的词。** 这是最简单直接的方法。

**优点：** 计算速度快，不需要额外的计算。
**缺点：** 容易陷入"局部最优"。有时候概率第二高的词在长远来看会导出更好的完整句子，但贪心搜索不会考虑这一点。

**比喻：** 贪心搜索就像走迷宫时"每次都选看起来最近的路"——你可能在每一步都选择了最近的距离，但最终走了一条弯路，反而更远。正确的做法应该是"全局规划"，虽然某一步看起来远一点，但总体路径更短。

**例子：** 假设在翻译 "The cat" 时：
- 第 1 步："The" -> "那个" (概率 0.6) vs "这" (概率 0.4)，贪心选"那个"
- 第 2 步：基于"那个 cat"，最佳翻译是"猫" (概率 0.3)，但"那只猫"整体概率是 0.6 x 0.3 = 0.18
- 如果第 1 步选"这"，第 2 步"只猫"概率可能是 0.7，整体概率是 0.4 x 0.7 = 0.28
- 贪心搜索选择了局部最优（"那个"），但全局来看"这只猫"更好

#### 4.6.2 Beam Search（束搜索）

**策略：在每一步保留 top-k 个最佳候选序列，同时探索多条路径。** Beam Search 是贪心搜索和穷举搜索之间的折中方案。

**工作原理：**
1. 第 1 步：选择概率最高的 k 个词，得到 k 个候选序列（k 就是"beam size"）
2. 第 2 步：对每个候选序列，生成所有可能的下一个词（k x 词表大小个候选），然后选择整体概率最高的 k 个
3. 重复以上步骤，直到所有候选序列都生成了结束符
4. 从所有完成的序列中选择总概率最高的一个

**具体例子（beam size = 3）：**
- 第 1 步：top-3 候选 = ["我"(0.5), "他"(0.3), "这"(0.2)]
- 第 2 步：每个候选扩展，选择 top-3 组合 = ["我爱"(0.25), "这爱"(0.14), "他爱"(0.12)]
- 第 3 步：继续扩展，最终选择总概率最高的完整序列

**比喻：** Beam Search 就像"派出 k 个探险队同时探路"。每个探险队（beam）独立地往前走一步，然后你从所有探险队中选出走得最好的 k 个继续前进。最终，到达终点的探险队中，总路程最短的就是最佳路线。

**Beam Size 的影响：**
- beam size = 1：等价于贪心搜索
- beam size = 4-10：常用的设置，平衡质量和速度
- beam size 太大：计算量增加，但质量提升递减

**Beam Search 的优缺点：**
- 优点：比贪心搜索质量更好，在翻译等任务上效果显著
- 缺点：输出可能比较"安全"和"平庸"，缺乏创造性；不适用于开放式生成任务（如对话、写作）

#### 4.6.3 Top-K 采样

**策略：从概率最高的 k 个词中随机采样。** Top-K 采样是一种随机性方法——每次生成时，先从概率分布中选出前 k 个最可能的词，丢弃其余词，然后将这 k 个词的概率重新归一化（使它们之和为 1），最后按概率随机选择一个。

**具体例子（k=5）：**
- Softmax 输出：["我":0.5, "他":0.2, "你":0.1, "她":0.1, "它":0.05, "我们":0.03, ...]
- 取 top-5：["我":0.5, "他":0.2, "你":0.1, "她":0.1, "它":0.05]
- 重新归一化：["我":0.531, "他":0.213, "你":0.106, "她":0.106, "它":0.053]
- 按概率随机选择（"我"被选中的概率最大，但不是 100%）

**比喻：** Top-K 采样就像"点菜时只看菜单前 5 道推荐菜"。你不会考虑排名第 100 的菜（质量太低），但也不是每次都点排名第一的菜（太单调）。你从前 5 道推荐菜中按推荐程度随机选择——既保证了质量，又有一定的变化性。

**k 值的影响：**
- k = 1：等价于贪心搜索（没有随机性）
- k = 50-100：常用的设置，兼顾质量和多样性
- k = 全部词表：纯随机采样，质量可能很差

**Top-K 的缺点：** 当概率分布非常"尖锐"（一个词的概率远高于其他词）时，k=50 可能仍然包含了太多低质量候选；当概率分布非常"平坦"（很多词的概率差不多）时，k=50 可能又不够。

#### 4.6.4 Top-P 采样（核采样，Nucleus Sampling）

**策略：从概率累计达到 p 的最小词集合中随机采样。** Top-P 采样（也叫 Nucleus Sampling）会动态调整候选词的数量，而不是固定为 k 个。

**工作原理：**
1. 将所有词按概率从高到低排序
2. 依次累加概率，直到累计概率达到 p（比如 0.9）
3. 只保留累计概率达到 p 的那些词，丢弃其余词
4. 在保留的词中按概率随机采样

**具体例子（p=0.9）：**
- 排序后的概率：["我":0.5, "他":0.2, "你":0.15, "她":0.1, "它":0.03, ...]
- 累计概率：0.5 -> 0.7 -> 0.85 -> 0.95（超过 0.9，停止）
- 保留前 4 个词：["我", "他", "你", "她"]
- 重新归一化后随机采样

**比喻：** Top-P 采样就像"自助餐时只拿前 90% 好吃的菜"。不管总共有多少道菜，你只拿"最好吃的那些"，使它们占总量的 90%。如果有一道菜特别好吃（概率很高），你可能只需要拿 3 道菜就够了；如果所有菜都差不多（概率均匀），你可能需要拿 20 道菜。

**Top-P vs Top-K：**
- Top-P 更灵活——候选词数量根据概率分布动态调整
- Top-K 是固定的——无论概率分布如何，候选词数量始终为 k
- 实践中 Top-P 通常比 Top-K 效果更好，因为它的自适应特性

#### 4.6.5 Temperature（温度参数）

**策略：调整概率分布的"尖锐程度"。** Temperature 是一个大于 0 的参数，在 Softmax 之前对分数进行缩放：

```
概率 = Softmax(分数 / 温度)
```

**Temperature 的影响：**
- 温度 = 1.0：正常概率分布
- 温度 < 1.0（如 0.3）：概率分布更"尖锐"，高概率词更突出，低概率词更小。输出更确定、更保守。
- 温度 > 1.0（如 1.5）：概率分布更"平坦"，高概率词和低概率词的差距缩小。输出更随机、更多样。
- 温度 -> 0：接近贪心搜索（只选概率最高的词）
- 温度 -> 无穷大：接近均匀随机采样

**比喻：** Temperature 就像是"创意阀门"。温度低 = 创意阀门关紧，每次都说最"安全"的话；温度高 = 创意阀门大开，每次都可能说出意想不到的话。

**实际应用中的组合：** 在 ChatGPT 等产品中，通常同时使用 Temperature 和 Top-P。典型的配置是 Temperature=0.7, Top-P=0.9——既有一定的确定性（不会说完全不相关的话），又有多样性（不会每次都说一样的话）。

#### 4.6.6 各策略对比总结

| 策略 | 随机性 | 质量 | 多样性 | 适用场景 |
|------|--------|------|--------|---------|
| 贪心搜索 | 无 | 中 | 低 | 简单任务 |
| Beam Search | 低 | 高 | 低 | 翻译、摘要 |
| Top-K 采样 | 中 | 中 | 中 | 对话、写作 |
| Top-P 采样 | 中 | 中 | 中 | 对话、写作（更优） |
| Temperature 控制 | 可调 | 可调 | 可调 | 配合采样使用 |

**比喻总结：**
- 贪心搜索 = 考试时"只写标准答案"——安全但无聊
- Beam Search = 考试时"同时写几份答案，选最好的"——质量高但缺乏个性
- Top-K = 考试时"从前 50 个知识点中随机选一个来答"——有趣但可能跑题
- Top-P = 考试时"从最相关的知识点中随机选一个来答"——平衡质量和趣味性
- Temperature = 调整"创新程度"的旋钮——低温保守，高温创新

## 5. 典型应用

### 5.1 三大家族：同一个 DNA，三种不同的"职业方向"

Transformer 诞生后，研究者们发现它的编码器和解码器可以独立使用，也可以组合使用。根据不同的任务需求，发展出了三个主要方向：

- **Encoder-Only（仅编码器派）**：只使用编码器部分，擅长"理解"类任务
- **Decoder-Only（仅解码器派）**：只使用解码器部分，擅长"生成"类任务
- **Encoder-Decoder（完整架构派）**：同时使用编码器和解码器，擅长"转换"类任务

**比喻：** 如果把原始 Transformer 比作一个"翻译公司"（编码器=编辑部，解码器=翻译部），那么：
- Encoder-Only 就像"只保留编辑部"——专注于阅读理解，不负责翻译
- Decoder-Only 就像"只保留翻译部"——不阅读原文，直接凭记忆和经验来写作
- Encoder-Decoder 就是"保留完整公司"——先理解再转换，适合翻译等任务

### 5.2 GPT 系列：只用了解码器（Decoder-Only）

#### 核心设计

GPT（Generative Pre-trained Transformer）系列由 OpenAI 开发，只使用了 Transformer 的**解码器部分**。它的工作原理非常简单：**给定前面的词，预测下一个词**。这就是所谓的"因果语言建模"（Causal Language Modeling）。

**为什么不需要编码器？** GPT 的核心能力是"文本生成"——你给它一个开头（prompt），它就一个词一个词地续写下去。这种任务只需要解码器的自回归生成能力，不需要编码器来"理解"一个单独的输入序列。解码器内部的掩码自注意力已经足以处理输入和生成之间的所有关系。

**比喻：** 如果说编码器是"阅读理解专家"，那 GPT 就是一个"即兴演讲者"——你给它一个话题开头，它就能滔滔不绝地讲下去。它不需要先仔细阅读一篇文章再做翻译，而是直接根据自己的"知识储备"来生成内容。

#### GPT 系列演进

| 模型 | 年份 | 参数量 | 关键突破 |
|------|------|--------|---------|
| GPT-1 | 2018 | 1.17 亿 | 首次将 Transformer 解码器用于预训练 |
| GPT-2 | 2019 | 15 亿 | 展示了"零样本学习"（Zero-shot Learning）能力——无需微调即可完成多种任务 |
| GPT-3 | 2020 | 1750 亿 | 引入"少样本学习"（In-Context Learning）——在提示中给几个例子就能完成新任务 |
| ChatGPT (GPT-3.5) | 2022 | 未公开 | 在 GPT-3 基础上通过"指令微调"和"人类反馈强化学习"（RLHF）大幅提升对话能力 |
| GPT-4 | 2023 | 据传约 1.8 万亿 | 支持多模态输入（文本+图像），推理能力大幅提升 |

**关键数据：** GPT-3 的训练使用了约 3000 亿个 token 的文本数据，训练成本估计超过 1200 万美元。GPT-4 据传使用了约 13 万亿个 token 的训练数据，训练成本估计超过 1 亿美元。

#### ChatGPT 是如何训练的？

ChatGPT 的训练分为三个阶段：

1. **预训练（Pre-training）：** 在海量文本数据上训练"预测下一个词"的能力。这个阶段赋予模型广泛的知识和语言能力。比喻：就像一个人读了整个互联网的所有内容——虽然他不一定理解所有东西，但他积累了大量的"知识碎片"。

2. **指令微调（Instruction Tuning）：** 在人工编写的"指令-回答"对上微调，让模型学会理解人类指令并给出合适的回答。比喻：就像一个人读完书之后，开始做练习题——学会按照要求来回答问题，而不是自顾自地说话。

3. **人类反馈强化学习（RLHF）：** 让人类对模型的多个回答进行排序，然后用强化学习训练模型生成更"符合人类偏好"的回答。比喻：就像老师给学生的多篇作文打分排序，告诉学生"这种写法更好"——学生逐渐学会写出让老师满意的文章。

### 5.3 BERT 系列：只用了编码器（Encoder-Only）

#### 核心设计

BERT（Bidirectional Encoder Representations from Transformers）由 Google 于 2018 年发布，只使用了 Transformer 的**编码器部分**。与 GPT 不同，BERT 的核心创新是**双向理解**——它可以同时利用一个词的左侧和右侧上下文来理解这个词的含义。

**为什么不需要解码器？** BERT 的核心能力是"文本理解"，而不是"文本生成"。它不需要逐词生成文本，只需要对输入文本进行"理解"和"编码"，然后输出每个词的上下文化表示，供下游任务使用。

**比喻：** 如果说 GPT 是一个"即兴演讲者"（擅长说），那 BERT 就是一个"精读分析专家"（擅长读）。你给它一篇文章，它能告诉你每个词在上下文中的精确含义、两句话是否相关、这段话的情感是正面还是负面——但它不会"说话"，不能生成新文本。

#### BERT 的训练方式：掩码语言建模（MLM）

BERT 使用了一种叫做**掩码语言建模**（Masked Language Modeling，简称 MLM）的预训练方法：随机遮盖输入中 15% 的 token，让模型预测被遮盖的 token 是什么。

**具体例子：**
```
原始句子：The cat sat on the mat
输入：The [MASK] sat on the mat
目标：预测 [MASK] 位置应该是 "cat"
```

**遮盖策略（对选中的 15% token）：**
- 80% 替换为 [MASK] 符号
- 10% 替换为随机 token
- 10% 保持不变

**为什么不全部替换为 [MASK]？** 因为在实际使用时（微调阶段），输入中不会有 [MASK] 符号。如果预训练时只看到 [MASK]，模型会学到一种偏差：只在 [MASK] 位置产生好的表示，而在正常 token 位置表现不佳。保留 10% 不变和 10% 替换为随机 token，可以缓解这个问题。

**比喻：** MLM 就像"完形填空"考试——给你一篇有空白处的文章，让你根据上下文填入正确的词。通过做大量的完形填空，BERT 学会了深入理解语言的上下文关系。

#### BERT 的模型配置

| 版本 | 层数 | 隐藏维度 | 注意力头数 | 参数量 |
|------|------|---------|-----------|--------|
| BERT-base | 12 | 768 | 12 | 1.1 亿 |
| BERT-large | 24 | 1024 | 16 | 3.4 亿 |

#### BERT 的典型应用

- **文本分类：** 情感分析（好评/差评）、垃圾邮件检测、新闻分类
- **命名实体识别（NER）：** 从文本中识别人名、地名、机构名等
- **问答系统：** 阅读理解式问答（给定文章和问题，从文章中找答案）
- **语义相似度：** 判断两句话是否表达相同的意思（搜索引擎的核心技术）
- **搜索引擎：** Google 搜索在 2019 年开始使用 BERT 来理解搜索查询的意图

### 5.4 T5 系列：用了完整的编码器+解码器（Encoder-Decoder）

#### 核心设计

T5（Text-to-Text Transfer Transformer）由 Google 于 2019 年发布，使用了 Transformer 的**完整编码器-解码器架构**。T5 的核心创新是**将所有 NLP 任务统一为"文本到文本"的格式**——无论什么任务，输入都是文本，输出也都是文本。

**T5 如何处理不同任务：**

| 任务 | 输入 | 输出 |
|------|------|------|
| 翻译 | "translate English to German: That is good" | "Das ist gut" |
| 摘要 | "summarize: 一篇很长的文章..." | 摘要文本 |
| 情感分析 | "sentiment: 这部电影太棒了！" | "positive" |
| 问答 | "question: 首都是哪里？ context: 北京是中国的首都" | "北京" |

**比喻：** 如果说 BERT 是一个"阅读理解专家"（只读不说），GPT 是一个"即兴演讲者"（只说不读），那 T5 就是一个"全能翻译官"——既能读又能写，把一种文本"转换"成另一种文本。翻译只是它最擅长的任务之一。

#### T5 的训练方式：跨度破坏（Span Corruption）

T5 使用了一种叫做**跨度破坏**（Span Corruption）的预训练方法：随机选择文本中的连续 token 片段（span），用特殊的哨兵 token 替换它们，然后训练模型恢复原始片段。

**具体例子：**
```
原始文本：Thank you for inviting me to your party last week
输入：Thank you <X> me to your party <Y> week
输出：<X> for inviting <Y> last <Z>
```

注意输出中多了 <Z>——因为原始文本有三个 span 被替换，但只有两个被显式替换为哨兵 token，第三个（"last"）需要模型自己发现并恢复。

**比喻：** Span Corruption 就像"猜词游戏"——你的朋友把一段话中的一些词替换成了编号占位符，你需要根据上下文猜出每个占位符原来是什么词。

#### T5 的模型配置

| 版本 | 参数量 | 层数 | 模型维度 | 注意力头数 |
|------|--------|------|---------|-----------|
| T5-Small | 6000 万 | 6 | 512 | 8 |
| T5-Base | 2.2 亿 | 12 | 768 | 12 |
| T5-Large | 7.7 亿 | 24 | 1024 | 16 |
| T5-3B | 30 亿 | 24 | 1024 | 16 |
| T5-11B | 110 亿 | 24 | 1024 | 16 |

### 5.5 三大家族对比总结

| 维度 | GPT（Decoder-Only） | BERT（Encoder-Only） | T5（Encoder-Decoder） |
|------|---------------------|---------------------|----------------------|
| 使用的部分 | 只有解码器 | 只有编码器 | 完整的编码器+解码器 |
| 核心能力 | 文本生成 | 文本理解 | 文本转换 |
| 训练方式 | 预测下一个词 | 掩码语言建模（完形填空） | 跨度破坏（猜词游戏） |
| 注意力类型 | 因果自注意力（只能看前面的词） | 双向自注意力（可以看前后所有词） | 编码器双向+解码器因果 |
| 输出方式 | 自回归生成（逐词输出） | 直接输出表示（不生成文本） | 自回归生成（逐词输出） |
| 典型应用 | ChatGPT、写作、对话、代码 | 搜索、分类、NER、情感分析 | 翻译、摘要、语音识别 |
| 代表模型 | GPT-3/4、LLaMA、Qwen、Mistral | BERT、RoBERTa、DeBERTa | T5、BART、Whisper |
| 代表参数量 | 175B-1.8T | 110M-1.5B | 60M-11B |

**比喻总结：**
- **GPT = 即兴演讲者。** 给它一个话题，它就能滔滔不绝地说。擅长"说"，不擅长"读"。ChatGPT、文心一言、通义千问都基于这种架构。
- **BERT = 阅读理解专家。** 给它一篇文章，它能深入理解每个词的含义。擅长"读"，不会"说"。Google 搜索、很多企业内部的文本分析系统使用这种架构。
- **T5 = 全能翻译官。** 既能读又能写，把一种文本转换成另一种文本。翻译、摘要、语音识别等"输入输出不同"的任务最适合这种架构。Google Translate 就基于 T5 的变体。

### 5.6 从论文到世界：Transformer 的影响力

**参数量增长：**
```
2017 Transformer:  6,500 万
2018 BERT:         3.4 亿    (增长 5x)
2020 GPT-3:        1,750 亿  (增长 270x)
2023 GPT-4:        约 1.8 万亿 (增长 10x)
```

**超越语言：** Transformer 的影响力早已超出自然语言处理领域：
- **计算机视觉：** ViT（2020）证明了 Transformer 也能处理图像——将图像切成小块当作"词"来处理
- **语音处理：** Whisper（2022）用 Encoder-Decoder 架构做语音识别，在 68 万小时音频上训练
- **多模态：** GPT-4V、Gemini 等模型同时处理文本、图像、音频
- **蛋白质预测：** AlphaFold2 使用类似 Transformer 的架构预测蛋白质结构
- **时间序列：** 用 Transformer 预测股票价格、天气等时序数据

**核心洞察：** Transformer 的成功不仅仅是因为它在语言任务上表现好，更因为它提供了一个**通用的信息处理框架**——只要你能把某种数据表示成"序列"的形式（文本是词的序列，图像是像素块的序列，音频是采样点的序列），就可以用 Transformer 来处理。这就是为什么 Transformer 被称为"AI 的通用引擎"。
