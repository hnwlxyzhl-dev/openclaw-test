# Transformer PPT 质检标准

## 幻灯片参数
- 尺寸: 13.333 x 7.5 英寸
- 宽屏比例 16:9

## 边界控制（最重要！）
- 所有元素: left + width ≤ 13.2
- 所有元素: top + height ≤ 7.3
- 底部安全区: top ≥ 6.8 保留给页脚
- 文本框宽度: ≤ 5 英寸（右文布局）
- 架构图区域宽度: ≤ 7.5 英寸（左图布局）

## 字体标准
- 封面标题: ≥ 36pt
- 章节标题: ≥ 24pt
- 正文: ≥ 10pt（根据内容量动态 10-14pt）
- 架构图标注: ≥ 10pt
- 目录项: ≥ 12pt
- 表格内容: ≥ 9pt

## 7维度检查（满分100分）

### 1. 文字不超画框（20分）— 最重要！
- 每个文本框的 left + width ≤ 13.2
- 每个文本框的 top + height ≤ 7.3
- 任何超框直接扣完20分

### 2. 架构图文字够大（15分）
- 所有标注 ≥ 10pt
- 关键组件名称 ≥ 12pt
- 在缩略图（50%）下仍可辨认

### 3. 箭头指向明确（10分）
- 箭头有明确起点和终点
- 箭头不重叠
- 数据流向清晰

### 4. 完整句子表达（15分）
- 不允许短句碎片
- 必须有形象比喻
- 必须有具体数字
- 必须通俗易懂

### 5. 布局合理占满（15分）
- 无大面积空白
- 图文比例合理（图60-70%，文30-40%）
- 元素对齐整齐

### 6. 内容详细通俗（15分）
- 非AI背景能看懂
- 每个概念都有比喻
- 数学公式有直觉解释
- 逻辑连贯

### 7. 图文紧密结合（10分）
- 文字解释图片
- 有编号对应关系
- 不是"图片+文字"的简单拼接

## 历史教训（必须避免！）
1. ❌ 文字超出画框 → 每个shape必须验证边界
2. ❌ 架构图坐标飞出 → 用合理的Inches值
3. ❌ 中文引号导致语法错误 → 只用ASCII引号
4. ❌ 同一主题拆多页 → 一个概念一页
5. ❌ 描述太简单 → 必须完整句子+比喻
6. ❌ 字体一刀切 → 根据内容量动态调整
7. ❌ 图片太小 → 架构图要足够大
8. ❌ 没有自我检查 → 生成后必须检查
