# Stable Diffusion PPT 质检标准（质检子代理必读）

> ⚠️ 这是基于真实项目血泪教训总结的质检标准，必须逐项检查，不可跳过！

---

## 📋 7维度检查清单（满分100分）

### 1️⃣ 文字不超画框（20分）⭐⭐⭐⭐⭐ 最重要！

**检查方法：** 用 python-pptx 读取每页每个文本框，检查所有文字是否在框内。

**历史教训：**
- v1: 5.5英寸高度 → 超框 ❌
- v2: 5.2英寸高度 + 12pt字体 → 仍超框 ❌
- 反复出现N次，用户极度不满："连文字不要超出范围都做不到"

**安全参数：**
- 文字框宽度 ≤ 3.8英寸（保守值）
- 文字框高度：5.4-5.8英寸
- 底部边距 ≥ 0.8英寸
- 左右边距各 ≥ 1英寸

**判定标准：**
- 全部文字都在框内 → 20分
- 有1-2处轻微溢出 → 10分
- 有3处以上溢出 → 0分

---

### 2️⃣ 图片/架构图文字够大（15分）

**历史教训：**
- 用户反馈："架构图本身的文字太小，看不清"
- 架构图上的标注字号必须 ≥18pt
- 箭头上的文字 ≥14pt

**检查方法：**
- 检查架构图/图片上的所有文字标注
- 图片中标题文字 ≥24pt
- 图片中正文文字 ≥18pt
- 图片中次要文字 ≥14pt

---

### 3️⃣ 箭头指向明确（10分）

**历史教训：**
- "箭头指向的目标框不清晰"

**检查方法：**
- 每个箭头都要有明确的目标框
- 箭头线条粗细 ≥3px
- 箭头起点和终点不能有歧义
- 使用不同颜色区分不同数据流

---

### 4️⃣ 完整句子表达（15分）

**历史教训：**
- ❌ 错误："Skip Connections（绿线）传递细节" — 太简单
- ❌ 错误："CLIP文本编码器" — 短句碎片
- ✅ 正确："Skip Connections（跳跃连接）就像给编解码器搭建了一座桥梁，让编码器阶段的重要细节信息能够直接传递给对应的解码器层，避免信息在压缩过程中丢失。比如64×64层的细节会直接传给64×64层。"

**检查标准：**
- 必须用完整、通俗易懂的句子
- 必须有形象比喻（"就像..."）
- 必须有具体数字（参数量、维度等）
- 非AI专业学生能看懂
- 不能只有名词+动词的短句

---

### 5️⃣ 布局合理占满（15分）

**黄金比例：**
- 图片占 60-70%（8.5×5.6-6.25英寸）
- 文字占 30-40%（3.8英寸宽）

**历史教训：**
- "图片太小（5.8英寸宽），大量空白区域"
- 必须占满整页，不留大量空白

**检查方法：**
- 图片+文字是否刚好占满整页
- 是否有大片空白区域
- 左右布局：图片左60%，文字右40%

---

### 6️⃣ 内容详细通俗（15分）

**检查标准：**
- 每个技术组件都有4个维度的说明：输入/输出/参数/作用
- 有具体数字（如"77×768维向量"、"123M参数"）
- 有形象比喻（如"VAE把512×512×3压缩成64×64×4，压缩48倍，就像ZIP压缩软件"）
- 面向非AI专业学生
- 不是堆砌术语

---

### 7️⃣ 图文紧密结合（10分）

**历史教训：**
- "图片和文字必须紧密结合"
- "必须有编号系统（1️⃣2️⃣3️⃣4️⃣）"
- "文字必须详细解释图片每个部分"

**检查方法：**
- 图片和文字是否有明确的对应关系
- 是否使用编号系统标注对应关系
- 文字是否解释图片中的每个组件
- 不能"图归图、文归文"

---

## 🚨 严重错误（一票否决，总分直接0分）

1. **同一主题拆成多页** — 一个技术组件必须在一页内讲完
2. **内容与目录不匹配** — 目录有的章节必须有对应页面
3. **中文用非微软雅黑字体** — 中文字体必须是微软雅黑
4. **输出格式不是 .pptx** — 必须是 PowerPoint 格式

---

## 📐 动态布局标准

根据内容量调整字体大小：

| 内容量 | 字体大小 | 布局策略 |
|--------|----------|----------|
| ≤8行 | 16pt大字体 | 宽松舒适，突出重点 |
| 9-15行 | 12pt中等字体 | 平衡美观与信息量 |
| 16-20行 | 11pt较小字体 | 稍紧凑，信息丰富 |
| ≥21行 | 10pt小字体 | 紧凑排列，信息密集 |

**字体规范：**
- 中文字体：微软雅黑
- 英文字体：Arial / Calibri
- 标题字体：微软雅黑 Bold
- 代码字体：Consolas / Source Code Pro

## 📝 历史成功经验（必须坚持）

1. **动态布局** — 内容少用大字，内容多用小字
2. **双代理质检** — 独立视角更容易发现问题
3. **严格边距** — 文字框宽度3.8英寸，字体10-16pt
4. **完整句子** — 有比喻、有数字、有细节
5. **图文并茂** — 图片60% + 文字40%，编号系统对应
6. **每页一个主题** — 不拆分
5. **生成后100%检查** — 不能偷懒跳过
