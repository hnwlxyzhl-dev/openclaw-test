## Fundamentals 50% 门槛机制（用户问）

- 门槛用 `report_count`（当季有有效 q_p 的股票数），不是 YoY/QoQ 的 match_count
- 代码：`strict_valid['report_count'] >= N_total * 0.5`
- `report_count` = 当季有多少只股票报了财报（有单季度利润数据）
- `yoy_match_count` / `qoq_match_count` 是结果标注，不是门槛条件

## A股 vs 美股（Global）架构差异（用户要求详细对比）

### 核心区别

1. **聚合路径数不同**：
   - A股（Index_PE_Push.py）：**只有1条路径**（forward-fill），用 `strict_set` 在聚合时过滤，同一个 `idx_q` 同时服务 PE 和 Fundamentals
   - 美股（Global_Index_PE.py）：**2条路径**。路径1（ffill）提供 TTM+cap_fraction → 用于 PE；路径2（strict）提供 YoY+QoQ → 覆盖路径1的值用于 Fundamentals

2. **50% 门槛的字段名不同**：
   - A股：`strict_count`（实际在 strict_set 中有当季报告的股票数）
   - 美股：`report_count`（当季有有效 q_p 的股票数）
   - 含义相同

3. **YoY/QoQ 过滤机制不同**：
   - A股：在 forward-fill 聚合循环中，用 `strict_report` mask 过滤（只在 strict_set 中的股票才参与加总）
   - 美股：路径1的 agg_quarter 不做 strict 过滤（所以 YoY/QoQ 可能混入 ffill 旧数据），路径2的 strict 聚合每只股票严格匹配 end_date

4. **数据源差异**：
   - A股：tushare `income` 累计值 → diff 得单季度；权重动态拉取；总市值 tushare 直接给
   - 美股：akshare `单季报` 已是单季度值；权重硬编码；总市值用腾讯快照×指数比率估算

5. **YoY/QoQ 计算方法**：两者相同（加总对比法，非加权平均）
6. **PB**：A股有（tushare total_equity），美股为 NaN（API 不提供）

### 相同点
- TTM 严格要求4个完整季度
- PE = total_mv × cap_fraction / TTM
- PEG = PE / Profit_YoY（严格值）
- eff_date = end_date + 1天（50% 门槛通过后）
