# 2026-06-20

## Hunter 会话卡死修复

**问题：** Hunter 卡在后台进程 `dawn-zephyr` 的轮询循环里，无法响应飞书消息。
- 症状：用户发消息，Hunter 无回复
- 原因：后台进程挂掉，但 Hunter 一直在 `process(poll)` 等待，3次轮询分别 600s/300s/300s 超时后继续等

**修复步骤：**
1. `sessions_history` 查看卡住会话的最后消息，确认为轮询死循环
2. 备份并重置磁盘上的会话文件（mv 到 `.reset.` 后缀）
3. `openclaw sessions cleanup --agent hunter --fix-missing --enforce` 清理 store
4. **关键：** 磁盘重置后 Gateway 内存仍缓存旧状态，`gateway restart` 超时
5. 通过 `sessions_send` 发新消息，触发 Gateway 重建会话 → 成功恢复

**教训：**
- 重置会话后必须验证 Gateway 内存状态是否同步
- `sessions_send` 可以强制触发会话重建，比 restart gateway 更轻量
- 排查 Agent 不响应问题：`sessions_list` → `sessions_history` → 检查最后消息是否有轮询死循环
