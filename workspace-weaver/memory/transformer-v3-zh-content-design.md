### 第3页：RNN/LSTM的三大痛点

**页面标题：** RNN/LSTM的三大痛点

**左侧——RNN顺序处理流程图：**
- 水平排列6个词方框：The → cat → sat → on → the → mat
- 每个方框之间用箭头连接，箭头上标注隐藏状态 h_t
- 用红色标注三个痛点位置：
  - 第1-2个词之间标注🔴"痛点①：串行计算，GPU利用率极低"
  - 第1-6个词之间画一条长弧线标注🔴"痛点②：长距离信息衰减（梯度消失）"
  - 最后一个词后画一个压缩箭头指向一个小方框（512维向量）标注🔴"痛点③：所有信息压缩到一个向量"
- 底部画一个虚线框代表LSTM，标注"LSTM通过门控缓解了痛点②，但痛点①③依然存在"

**右侧——三段详细分析：**

① **顺序处理的效率陷阱：** RNN必须一个词一个词按顺序处理，每个词的处理都要依赖前一个词的隐藏状态h_t。这意味着即使GPU有数千个并行计算单元，RNN也只能串行使用其中一个。在实际训练中，处理一个包含1000个词的句子，RNN需要1000个顺序步骤。这就像雇了1000个工人搬砖，却规定每次只能1个人搬，效率浪费了99.9%。LSTM虽然引入了门控机制缓解了梯度消失，但本质上的顺序处理限制完全没有改变。

② **长距离遗忘问题：** 当RNN处理到句子末尾时，通过隐藏状态传递的早期信息已经被数百次矩阵乘法稀释，这种现象叫做梯度消失。具体来说，每一步的隐藏状态更新为h_t = tanh(W·h_{t-1} + U·x_t)，如果W的特征值小于1，信息会指数级衰减。在100步之后，早期信息可能只剩下10^-30的权重，几乎完全丢失。LSTM通过门控机制（遗忘门、输入门、输出门）可以选择性地保留信息，但在处理超过100词的长文本时，遗忘问题仍然严重。

③ **信息瓶颈：** RNN将整个输入序列压缩成一个固定长度的向量（通常是256维或512维），然后从这个向量生成输出。但一个512维的向量能表达的信息量极其有限——试想一下，用100个汉字的摘要来概括一本500页的书籍，再根据这个摘要回答关于书中任意细节的问题，这是不可能完成的任务。信息瓶颈直接限制了模型处理长文本和理解复杂语义关系的能力。

### 第4页：核心突破——自注意力机制

**页面标题：** 核心突破——自注意力机制

**左侧——注意力可视化图：**
- 水平排列10个词：The / cat / sat / on / the / mat / and / it / was / happy
- "it"用蓝色高亮框
- 从"it"画不同粗细的连线到其他词：
  - "it" → "cat"：最粗线（蓝色），标注"注意力权重 0.6"
  - "it" → "mat"：次粗线，标注"0.2"
  - "it" → "sat"：中等线，标注"0.08"
  - "it" → "the"：细线，标注"0.03"
  - "it" → "was"：最细线，标注"0.05"
  - 其他词画虚线（权重很小）
- 底部画一个小的注意力热力图（10×10矩阵，"it"行用颜色深浅表示权重分布）

**右侧——三段详细分析：**

① **从排队发言到圆桌会议：** RNN像排队发言，每个人只能听到前面一个人说的话，信息要经过层层传递。Transformer像圆桌会议，每个人可以同时看到所有人，任何两个人之间都可以直接交流。这个核心区别解决了RNN的全部三个问题：并行化（所有人同时发言）解决顺序处理，直接交流解决长距离遗忘，不压缩信息解决信息瓶颈。

② **注意力权重如何工作：** 以上面的句子为例，当模型处理"it"这个词时，自注意力机制会让"it"与句子中的每个其他词计算相关性得分。"cat"得分最高（0.6），因为"it"作为代词最可能指代"cat"这个名词；"mat"得分次高（0.2），因为"mat"也是一个可能的指代对象；而"was"得分很低（0.05），因为情态动词与代词指代无关。最终"it"的表示向量是所有词向量的加权平均：0.6×向量(cat) + 0.2×向量(mat) + 0.05×向量(was) + ... 这样"it"就"知道"自己指的是"cat"了。

③ **数学本质：** 自注意力的核心计算是Q×K^T（查询×键的转置），本质上是在高维空间中计算两个向量之间的点积（余弦相似度的一种形式）。点积越大，两个词的语义关联越强。这个操作可以一次性计算所有词对之间的关联度，形成T×T的注意力矩阵，完全并行，无需顺序处理。

### 第5页：架构总览——左边理解，右边生成

**页面标题：** 架构总览——左边理解，右边生成

**左侧——完整Encoder-Decoder架构图（占页面60%宽度，高度约5.5英寸）：**

  - 用花括号括起来，标注"编码器：理解输入"
- **右侧解码器×6大框（红色边框#E74C3C）：**
  - 内部展示：Masked Self-Attention（蓝色） → Add & LayerNorm → Cross-Attention（紫色#9B59B6） → Add & LayerNorm → FFN（橙色） → Add & LayerNorm
  - 旁边标注"×6层"
  - 用花括号括起来，标注"解码器：生成输出"
- **关键连接线：**
  - 从编码器画两条线（K、V）到解码器的Cross-Attention，标注"编码器输出作为K、V"
- **输出区：** Linear（512×37000） → Softmax → 输出概率分布
- **维度标注：** 各层旁标注512维、8头×64维、2048维

**右侧——四段详细分析：**

① **编码器=编辑部：** 编码器的任务是深度理解输入文本。它由6个相同的层堆叠而成，每层包含两个子层——多头自注意力和前馈网络，每个子层都有残差连接和层归一化。数据在编码器中流动时，每个词的信息会与其他所有词的信息不断融合，经过6层处理后，每个词的512维表示已经蕴含了整个句子的上下文信息。编码器的参数量约3450万（6层×约575万/层）。

② **解码器=翻译部：** 解码器的任务是逐步生成输出。它也有6层，但比编码器多了一个子层——交叉注意力（Cross-Attention）。这个子层让解码器能够"查阅"编码器的输出，相当于带着问题去查资料。此外，解码器的自注意力使用因果掩码（Causal Mask），确保每个位置只能看到之前的词，不能偷看未来的答案。

③ **数据流全景：** 输入文本经过分词（如BPE子词分词）→ 词嵌入（37000×512矩阵查表，将每个词映射为512维向量）→ 位置编码（用sin/cos函数注入位置信息）→ 编码器6层处理（每层都在做"交流+提炼"）→ 编码器输出 → 解码器6层处理（每层多一步"查资料"）→ 线性变换（512维→37000维）→ Softmax（转为概率分布）→ 取概率最高的词作为输出。

④ **关键数字：** 论文原始配置——编码器6层、解码器6层、512维、8个注意力头、前馈网络2048维、参数总量6500万、训练数据450万英德句对、8块P100 GPU训练12小时、WMT 2014英德翻译BLEU 28.4。

**右侧——四段详细分析：**

① **多头自注意力=圆桌会议的8个讨论组：** 单头注意力只能从一个角度分析词之间的关系，而多头注意力使用8个独立的注意力头，每个头学习不同的关系模式。比如，头1可能专注于语法关系（主语-谓语）、头2关注语义相似性（同义词）、头3关注位置关系（相邻词）、头4关注指代消解（代词-名词）等。每个头独立计算Q、K、V（各64维），得到8个不同的注意力矩阵，最后将8个结果拼接起来（8×64=512维），经过一次线性变换融合。

② **残差连接=高速公路匝道：** 在深度神经网络中，梯度需要通过链式法则从输出层反向传播到输入层。当网络层数很多时（如Transformer的6层），梯度会不断乘以小于1的权重矩阵，导致指数级衰减（梯度消失）。残差连接通过将层的输入直接加到输出上（output = input + sublayer(input)），为梯度提供了一条"高速公路匝道"——即使中间层的权重很小，梯度也可以通过加法路径无损传播到任意层。这是Transformer能够成功堆叠多层的关键技术之一。

③ **层归一化=标准化体检：** 层归一化对每个词的512维向量独立做标准化操作：减去均值、除以标准差，再经过可学习的缩放和偏移参数。这确保了无论输入数值的范围如何，每一层的输入都在合理的数值范围内。没有归一化，随着层数增加，数值可能指数级增长或衰减，导致训练不稳定甚至无法收敛。层归一化的计算量为O(d)，相比批量归一化，不依赖批次大小，在序列长度变化时更稳定。

④ **前馈网络=独立思考空间：** 每个位置的前馈网络是一个两层的全连接网络：FFN(x) = ReLU(xW₁ + b₁)W₂ + b₂。第一层将512维扩展到2048维（4倍扩展），第二层再压缩回512维。中间使用ReLU激活函数（max(0, x)）引入非线性。有趣的是，前馈网络的参数量（512×2048×2 ≈ 210万）比自注意力（约100万）大2倍，是整个Transformer中计算量最大的组件。前馈网络的作用类似于给每个词提供"独立思考"的空间——注意力负责词之间的交流，FFN负责每个词自身的深层特征提取。

**左侧——解码器单层结构图：**

- 从左侧画"编码器输出(K,V)"箭头指向Cross-Attention，用紫色虚线标注
- 用颜色区分三个子层：蓝色（掩码自注意力）、紫色（交叉注意力）、橙色（FFN）

**右侧——三段详细分析：**

① **掩码自注意力=考试不能偷看后面的题：** 与编码器的自注意力不同，解码器的自注意力使用因果掩码（Causal Mask）。具体实现是将注意力分数矩阵的上三角部分（对应未来位置）设为负无穷大(-∞)，经过Softmax后变为0，使模型在处理每个词时只能看到自己和之前的词。例如处理第3个词"爱"时，只能看到[START, 我, 爱]，看不到后面的"你"。这在训练和推理中都必须严格执行，否则模型会"作弊"——训练时如果能看到答案，推理时没有答案就会崩溃。

② **交叉注意力=带着问题去查资料：** 交叉注意力是解码器独有的子层，也是编码器和解码器之间唯一的"桥梁"。它的Q（查询）来自解码器的上一层输出，代表"我在翻译，需要从原文中找到相关信息"；而K（键）和V（值）来自编码器的最终输出，代表"编辑对原文每个词的深度理解"。计算过程与自注意力完全相同：Q×K^T → 缩放 → Softmax → ×V。例如，当解码器正在生成"爱"这个词时，Query可能编码了"需要找一个情感动词"，通过与编码器的Key逐一比对，发现"love"的Key匹配度最高（注意力权重0.7），于是大量提取"love"的Value信息来辅助生成"爱"。

③ **前馈网络：** 与编码器完全相同，512→2048→ReLU→512。解码器的6层结构为：掩码自注意力+残差归一化 → 交叉注意力+残差归一化 → 前馈网络+残差归一化，每层共3个子层+3个残差归一化。相比编码器每层2个子层，解码器多出的交叉注意力子层是连接编码器输出的唯一通道，也是"理解→生成"信息流转的关键节点。

- 两者之间用双向箭头连接，标注"交叉熵损失"

**下方——三段详细分析：**

① **训练=开卷考试：** 训练阶段的核心特点是"所有位置并行计算"。对于目标序列[我, 爱, 你, END]，解码器同时看到完整输入[START, 我, 爱, 你]（Teacher Forcing策略），并行预测每个位置的下一个词。这就像开卷考试——每道题都能看到前面所有题的正确答案作为参考。这种并行化是Transformer训练效率远超RNN的根本原因：GPU可以同时处理序列中的所有位置，充分利用并行计算能力。

② **训练目标=学会预测下一个词：** 模型的唯一训练目标是：给定前面的词，预测下一个词是什么。具体来说，在编码器处理完原文后，解码器需要根据[START]预测"我"，根据[START, 我]预测"爱"，根据[START, 我, 爱]预测"你"，根据[START, 我, 爱, 你]预测"<END>"。每个位置的预测误差（交叉熵损失）会被平均，然后通过反向传播更新模型参数。经过足够多的训练样本，模型就能学会语言的统计规律。

③ **编码器和解码器的分工：** 编码器只处理输入（源语言），每批训练中编码器只运行一次，其输出（K、V向量）会被缓存供解码器的每一层使用。解码器处理目标序列，在每层中通过交叉注意力"查阅"编码器的输出。这种分离设计使得编码器专注于"理解"，解码器专注于"生成"，各司其职。

**左侧——嵌入+位置编码流程图（详细）：**

           │           │          │
           └───────────┼──────────┘
                       ▼

```

**右侧——四段详细分析：**

① **分词=把句子切成最小单元：** Transformer使用BPE（Byte Pair Encoding）子词分词方法。BPE从字符级别开始，反复合并语料中出现频率最高的字符对，直到达到预设的词表大小（通常30000-50000个）。例如，"unaffordable"可能被分为["un", "aff", "ord", "able"]。BPE的优势在于：可以处理任何未见过的词（OOV问题），常见词保持完整（如"the"），罕见词被拆分为子词片段（如"un"+"aff"+"ord"+"able"）。原始论文使用37000大小的词表。

② **嵌入矩阵=超级字典：** 词嵌入矩阵E的大小为V×d = 37000×512，共约1890万个参数，占模型总参数的30%。每个词在矩阵中对应一个512维的行向量，这个向量编码了词的语义信息。例如，"猫"和"狗"的向量会比较接近（都是动物），"国王"和"女王"的向量差异方向与"男人"和"女人"类似。这些语义关系不是人工设计的，而是模型通过大量数据训练自动学到的。嵌入矩阵是整个模型中最大的单个参数块。

③ **位置编码=给每个词贴座位号：** 自注意力机制本身没有位置概念——它同时处理所有词，不知道词的顺序。为了让模型区分"狗咬人"和"人咬狗"，必须显式注入位置信息。Transformer使用正弦/余弦函数生成位置编码：PE(pos, 2i) = sin(pos/10000^(2i/d))，PE(pos, 2i+1) = cos(pos/10000^(2i/d))。其中pos是位置索引，i是维度索引，d是嵌入维度(512)。这种设计的巧妙之处在于：不同位置的编码是唯一的，且可以表示相对位置关系。更重要的是，位置编码是固定的（不可学习的），不增加任何可训练参数。

④ **为什么位置编码如此设计：** 使用三角函数而非可学习的位置向量有几个重要原因：(1)三角函数可以将任意长度的位置映射到[-1,1]范围内，不会因为序列过长而数值爆炸；(2)不同频率的sin/cos函数使得模型可以自然地捕捉不同尺度的相对位置关系（相邻词 vs 跨句词）；(3)论文发现可学习的位置编码效果并不比固定编码好，说明这种设计已经足够。最终每个词的输入=词嵌入（语义信息）+ 位置编码（位置信息），两者在同一维度上直接相加。

### 第10页：训练·QKV矩阵生成

**页面标题：** 训练·QKV矩阵生成

**左侧——QKV生成详细流程图：**

- 三个分支用不同颜色区分（蓝/橙/绿）
- 底部画图书馆搜索的类比图示（搜索框 → 书架标签 → 打开书本）

**右侧——四段详细分析：**

① **Query=我在找什么信息：** 对于每个词，Query向量编码了"我需要寻找什么信息"。例如，当模型处理代词"it"时，"it"的Query向量会编码"我需要找到一个名词作为指代对象"。Query是通过将词向量乘以权重矩阵W_Q得到的：q = xW_Q，其中x是512维词向量，W_Q是512×64的权重矩阵，q是64维的Query向量。每个注意力头有自己独立的W_Q，所以8个头会生成8个不同的Query。

② **Key=我有什么特征：** Key向量编码了每个词的"身份特征"。例如，名词"cat"的Key向量会编码"我是名词、我是动物、我是单数、我是可数"等特征。当其他词的Query与"cat"的Key做点积时，如果Query在寻找一个"名词"，点积就会很高。Key的计算方式与Query完全相同：k = xW_K，W_K也是512×64的矩阵。

③ **Value=我的实际内容：** Value向量是每个词要传递给其他词的"实际内容"。在注意力计算的最后一步，每个词的输出是所有Value向量的加权平均（权重由Q×K^T的Softmax结果决定）。例如，如果"it"对"cat"的注意力权重是0.6，那么"it"的输出中就包含0.6倍的"cat"的Value信息。Value的计算：v = xW_V，W_V也是512×64的矩阵。

④ **8个头=8套独立视角：** 每个注意力头有自己独立的W_Q、W_K、W_V矩阵（各512×64），所以8个头总共需要4×512×64×8 = 约100万个参数。每个头独立计算自己的Q、K、V和注意力矩阵，捕捉不同类型的关系。最后将8个头的64维输出拼接成512维（concat），再经过一次线性变换融合。这确保了模型能同时从语法、语义、指代、位置等多个角度分析文本。

### 第11页：训练·自注意力5步计算

**页面标题：** 训练·自注意力5步计算

**左侧——5步垂直流程图（每步一个方框，高度约0.7英寸）：**

```
步骤① ┌─────────────────────────────────────┐ (蓝色)
      │ X → W_Q/W_K/W_V → Q, K, V          │
      │ "所有词同时计算，完全并行"             │
      └────────────────┬────────────────────┘
                       ▼
步骤② ┌─────────────────────────────────────┐ (浅蓝色)
      │ Q × K^T → 分数矩阵                   │
      │                                     │
      │  旁边画3×3注意力矩阵示意：             │
      │     词A  词B  词C                    │
      │  A [0.8  0.1  0.1]                  │
      │  B [0.2  0.6  0.2]                  │
      │  C [0.1  0.3  0.6]                  │
      └────────────────┬────────────────────┘
                       ▼
步骤③ ┌─────────────────────────────────────┐ (黄色#F39C12)
      │ 分数 ÷ √d_k (√64 = 8)              │
      │ "防止梯度饱和"                        │
      └────────────────┬────────────────────┘
                       ▼
步骤④ ┌─────────────────────────────────────┐ (绿色)
      │ Softmax → 概率矩阵                   │
      │ "每行和 = 1"                         │
      └────────────────┬────────────────────┘
                       ▼
步骤⑤ ┌─────────────────────────────────────┐ (紫色#9B59B6)
      │ 概率 × V → 输出向量                   │
      │ "融合所有词的信息"                     │
      └─────────────────────────────────────┘
```

- 5个方框用不同颜色区分，步骤编号用圆形标注
- 步骤②旁画3×3注意力矩阵的示意图
- 步骤③用黄色突出标注"√64=8"

**右侧——五段详细分析（每步一段）：**

**步骤①：生成Q、K、V矩阵。** 输入序列中每个词的512维向量X，分别乘以三个权重矩阵W_Q、W_K、W_V（各512×64），得到64维的Q、K、V向量。对于长度为T的序列，得到三个矩阵Q(T×64)、K(T×64)、V(T×64)。注意：这一步对序列中所有词同时计算，完全并行，没有任何顺序依赖。

**步骤②：计算注意力分数矩阵。** 将Q与K的转置相乘，得到T×T的分数矩阵：Score = QK^T。矩阵中每个元素Score[i,j]表示第i个词对第j个词的"关注程度"。例如，Score["it", "cat"]可能=12.5（高度关注），Score["it", "the"]可能=2.1（关注度低）。这本质上是在高维空间中计算余弦相似度。

**步骤③：缩放。** 将分数矩阵除以√d_k = √64 = 8。这一步至关重要——当维度d_k较大时，Q和K的点积的方差也会很大（如果Q和K的各分量独立且方差为1，则点积方差为d_k），导致Softmax函数进入饱和区，梯度接近0。除以√d_k将方差归一化为1，确保Softmax的梯度不会消失。这就像考试分数从百分制转换为十分制，保持分数的相对大小不变，但避免极端值。

**步骤④：Softmax归一化。** 对分数矩阵的每一行做Softmax操作，将分数转换为概率分布。Softmax(x_i) = e^(x_i) / Σe^(x_j)。转换后每行的和恰好等于1。例如，"it"对["The","cat","sat","on","the","mat"]的注意力权重可能是[0.02, 0.60, 0.05, 0.01, 0.03, 0.29]，其中"cat"最高（0.60）。Softmax的一个重要性质是：它会放大最大值但不会将其他值完全置零（与argmax不同），这使得每个词都能获得一些信息。

**步骤⑤：加权求和。** 将Softmax概率矩阵与Value矩阵相乘：Output = Attention_weights × V。每个词的输出是其自身和其他所有词Value向量的加权平均。以"it"为例，输出 = 0.02×V("The") + 0.60×V("cat") + 0.05×V("sat") + 0.01×V("on") + 0.03×V("the") + 0.29×V("mat")。经过这一步，"it"的表示向量中已经融合了"cat"的大量信息——模型"知道"了"it"指代的是"cat"。

### 第12页：训练·残差连接与层归一化

**页面标题：** 训练·残差连接与层归一化

**左侧——编码器单层数据流详图（带维度标注和旁路）：**

- 每个加号"⊕"旁标注公式

**右侧——四段详细分析：**

① **残差连接的数学原理：** 残差连接的公式为Output = LayerNorm(x + SubLayer(x))，其中x是层的输入，SubLayer是注意力或FFN。关键在于x被直接加到SubLayer的输出上。从梯度的角度看：∂Loss/∂x = ∂Loss/∂Output × (∂SubLayer/∂x + 1)。加号后面的"1"意味着无论SubLayer的梯度多小，总梯度中至少有一个"1"的贡献，梯度永远不会消失。这就是残差网络（ResNet）能训练100+层网络的核心原因。

② **层归一化的实现细节：** 层归一化的计算公式为y = γ × (x - μ) / σ + β，其中μ是均值，σ是标准差，γ（缩放参数）和β（偏移参数）是可学习的。对512维向量做归一化：先计算该向量的均值μ = mean(x_1,...,x_512)和标准差σ = std(x_1,...,x_512)，然后标准化为z_i = (x_i - μ)/σ，最后通过γ和β恢复表达能力。γ和β各512维，共1024个可学习参数/每层。

③ **为什么需要归一化：** 深度网络中的数值稳定性是一个核心挑战。没有归一化时，随着层数增加，各层的输出数值范围可能剧烈变化——某一层的输出可能在[-0.01, 0.01]范围内，下一层突然变成[-100, 100]。这种数值不稳定会导致梯度爆炸或消失，使训练完全无法进行。层归一化确保每层的输入都在标准化的数值范围内（均值为0，标准差为1），极大提高了训练的稳定性和收敛速度。

④ **一层完整数据流：** 输入(512维) → 注意力层(信息交流) → +残差 → 归一化(数值稳定) → FFN(特征提取) → +残差 → 归一化 → 输出(512维)。这个过程在编码器和解码器中重复6次。每经过一层，词的表示向量就变得更加丰富——不仅包含原始语义信息，还融合了更多上下文信息，并经过非线性变换提取了更高级的特征。

**右侧画交叉注意力示意：**
```
  编码器输出 (紫色)
  ┌─────────────────────┐
  │ K_enc │ V_enc       │  ← 来自编码器最终输出
  └───┬─────┬───────────┘
      │     │
      ▼     │
  ┌─────────────────────┐
  │  Q_dec × K_enc^T    │  ← Q来自解码器
  │  → Softmax → ×V_enc │
  └─────────────────────┘
```

**右侧——三段详细分析：**

① **因果掩码的工作原理：** 因果掩码是一个下三角矩阵M，其中M[i,j]=0当i≥j（可以看），M[i,j]=-∞当i<j（不能看）。注意力分数矩阵加上掩码后：Score_masked = Score + M。对于被禁止的位置，分数变成-∞，经过Softmax后e^(-∞)=0，完全不可见。以4×4为例，处理"爱"时（第3行），只能看到[START, 我, 爱]，"你"的注意力权重为0。这确保了严格的自回归特性：第t步的预测只依赖于第1到t个位置的信息。

② **为什么训练也需要掩码：** 这是一个常见的疑问——训练时我们已经有完整的目标序列了，为什么还要限制模型只能看到前面的词？答案是：为了保证训练和推理行为一致。如果训练时不加掩码，模型会学会"偷看"后面词的信息来辅助预测，形成依赖。但推理时没有后面的词可供偷看，模型的预测质量会急剧下降。这种训练-推理分布不匹配叫做"暴露偏差"（Exposure Bias），是序列生成任务中的经典难题。因果掩码从设计上杜绝了这个问题。

③ **交叉注意力的计算细节：** 交叉注意力的Q来自解码器的掩码自注意力输出，维度为T_dec×64；K和V来自编码器的最终输出，维度为T_enc×64。计算过程：Score = Q_dec × K_enc^T（T_dec × T_enc矩阵），然后缩放、Softmax，最后乘以V_enc。注意：Q和K的序列长度可以不同（源语言和目标语言长度通常不一样），这不像自注意力是T×T的方阵。交叉注意力使得解码器的每个位置都能"关注"编码器输出中的任意位置，实现源语言到目标语言的信息传递。

### 第14页：训练·损失函数——交叉熵

**页面标题：** 训练·损失函数——交叉熵

**上方——损失计算流程图：**

```
解码器最终输出 (512维)
        │
        ▼
┌──────────────────────────────────┐
│  线性变换 (512×37000权重矩阵)     │  ← 输出投影
│  logits = h·W_out + b_out        │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  Softmax → 预测概率分布 (橙色)    │  ← 37000维，所有概率和=1
│  p(i) = e^(logit_i) / Σe^(logit_j) │
└──────────────┬───────────────────┘
               │
               ▼
   预测概率 (橙色) ←────对比────→ 真实目标词 (红色)
               │                      │
               └──────────┬───────────┘
                          ▼
               ┌──────────────────────────┐
               │ 交叉熵损失                 │
               │ Loss = -log(p(正确词))     │  ← 红色高亮
               │ = -log(0.95) ≈ 0.05      │
               └──────────────┬───────────┘
                              │
                              ▼
               ┌──────────────────────────┐
               │  反向传播 (灰色箭头)        │
               │  ∂Loss/∂θ → 更新所有参数   │
               │  θ_new = θ_old - lr·∇θ   │  ← 灰色齿轮
               └──────────────────────────┘

  底部标注："Loss = -(1/T) × Σ log(p(正确词_t))，T是序列长度"
```

**下方——四段详细分析：**

① **输出层：线性变换+Softmax。** 解码器最后一层输出的512维向量需要映射到词表空间（37000个词）。这是通过一个大的线性变换（也叫"输出投影"）实现的：logits = hW_out + b_out，其中h是512维，W_out是512×37000的权重矩阵，logits是37000维的原始分数（logits）。然后通过Softmax将logits转换为概率分布：p(i) = e^(logit_i) / Σe^(logit_j)，所有37000个概率之和=1。概率最高的词就是模型认为最应该在当前位置出现的词。

② **交叉熵损失=衡量预测与真实答案的距离。** 交叉熵的公式为Loss = -log(p(正确词))。如果模型对正确词的预测概率p=0.99，Loss = -log(0.99) ≈ 0.01（很低，模型很有信心且正确）；如果p=0.01，Loss = -log(0.01) ≈ 4.6（很高，模型几乎猜错了）。训练目标是让Loss尽可能小。对于整个序列，总Loss是所有位置的平均：Loss = -(1/T)Σlog(p(正确词_t))。训练初期（随机初始化），Loss ≈ log(37000) ≈ 10.5；训练充分后，Loss通常降到1.5-3.0。

③ **反向传播：** 给定Loss，计算它对每个参数θ的偏导数∂Loss/∂θ。由于Transformer有6500万个参数，这是一个非常庞大的梯度计算过程。PyTorch等深度学习框架通过自动微分（Autograd）高效计算梯度。梯度计算完成后，使用优化器（原始论文使用Adam优化器）更新参数：θ_new = θ_old - lr × gradient，其中lr是学习率。原始论文使用了自定义的学习率调度：前4000步线性增温到峰值，之后按步数的-0.5次方衰减。

④ **训练配置细节：** 原始Transformer的训练配置为——WMT 2014英德翻译数据集，450万句对，批量大小25000个token，学习率峰值0.0003，Adam优化器(β₁=0.9, β₂=0.98, ε=10⁻⁹)，标签平滑ε_ls=0.1，dropout率0.3（训练时随机丢弃30%的神经元），8块P100 GPU训练12小时（30万步），最终BLEU得分28.4（当时SOTA）。

  位置3: 基于"他不"预测"是"(p=0.2) ← 完全偏离

═══════════════════════════════════════════════════════════════════
```

**右侧——四段详细分析：**

① **Teacher Forcing的原理：** Teacher Forcing是指在训练时，解码器的输入不是模型自己的预测，而是真实的目标序列。具体来说，当预测第t+1个词时，解码器的输入是[真实词_1, 真实词_2, ..., 真实词_t]，而不是[预测词_1, 预测词_2, ..., 预测词_t]。这就像开卷考试——每道题都能看到前面所有题的标准答案。好处是：每一步的输入都是正确的，模型可以在正确的基础上学习，不会因为前面的错误预测而偏离方向。

② **Teacher Forcing的巨大优势——并行化：** 这是最关键的优势。因为每一步的输入都是已知的真实答案，所有位置的预测可以完全并行计算。GPU可以同时处理序列中的所有位置，充分利用并行计算能力。相比之下，如果不使用Teacher Forcing，每一步都必须等上一步的预测完成才能开始（串行），训练速度会慢几百倍。这是Transformer训练效率远超自回归模型的关键原因。

③ **暴露偏差——Teacher Forcing的核心缺陷：** 训练时看正确答案，推理时看自己的输出，两者分布不一致。就像学生练习时有标准答案参考，考试时完全靠自己——练习中的高分不一定能转化为考试中的高分。具体来说，训练时模型从未见过"错误的输入"，一旦推理时自己的预测出错，后续预测的质量会急剧下降（误差累积）。这个问题在长序列生成时尤为严重。

④ **缓解策略：** Scheduled Sampling（Bengio et al., 2015）以概率p使用真实答案，以概率(1-p)使用模型预测，p从1.0逐渐降低到0.0，让模型逐步适应自己的输出。此外，还有Professor Forcing（用GAN训练判别器区分训练/推理行为）等方法。但在实践中，Teacher Forcing仍然是主流策略，因为它的训练效率优势太大。

### 第16页：推理阶段总览——闭卷考试

**页面标题：** 推理阶段总览——闭卷考试

**上方——自回归生成步骤图（4个Step纵向排列，每个Step约0.6英寸高）：**

```
┌─────────────────────────────────────────────────┐
│  编码器 (蓝色)                                   │
│  Input: "I love you" → 编码器×6 → 输出缓存(K,V) │
│  ⚡ 只运行一次 → 输出缓存                          │
└─────────────────────────────────────────────────┘
                    │
                    ▼
  ┌─────────────────────────────────────────────────┐

  │              ↓ 串行箭头                          │
  │ Step 2: [START, 我] → 解码器×6 → "爱" (p=0.78)  │
  │              ↓ 串行箭头                          │
  │ Step 3: [START, 我, 爱] → 解码器×6 → "你" (p=0.91)│
  │              ↓ 串行箭头                          │
  │ Step 4: [START, 我, 爱, 你] → 解码器×6 → "<END>" (p=0.96)│
  └─────────────────────────────────────────────────┘

  ══════════════════════════════════════════════════
       最终输出：我爱你 （橙色大字）
  ══════════════════════════════════════════════════
```

- 编码器用蓝色框，标注"只运行一次→输出缓存"
- 解码器每步用绿色框，标注串行箭头"逐词生成"
- 概率标注在每个输出词旁

**右侧——三段详细分析：**

① **推理=闭卷考试：** 与训练的"开卷考试"不同，推理阶段模型必须完全靠自己之前生成的词来预测下一个词。第一步只有[START]，模型凭"空"生成第一个词"我"；然后将"我"加入输入序列，生成"爱"；再加入"爱"，生成"你"；最后生成"<END>"标记停止。每一步都依赖前一步的正确性——如果第一步生成了"他"而不是"我"，后续的预测可能完全不同。这就像闭卷考试，第一道题做错了可能影响后面所有题的思路。

② **编码器只需运行一次：** 这是推理效率的关键优化。编码器处理输入"I love you"后，输出6层编码的向量表示被完整缓存（K和V向量），在整个生成过程中反复使用。解码器的每一层在计算交叉注意力时，都会查阅这份缓存，无需重新计算编码器的输出。这使得编码器的计算成本只发生一次，而与生成序列的长度无关。

③ **解码器的串行瓶颈：** 与训练的完全并行不同，推理时解码器必须逐词串行生成——生成第1个词后才能生成第2个词，以此类推。生成一个包含100个词的句子需要100次前向传播，而训练时只需要1次。这就是为什么Transformer的推理速度远慢于训练效率。KV缓存是缓解这个瓶颈的主要技术（下一页详述）。

### 第17页：推理·自回归生成与KV缓存

**页面标题：** 推理·自回归生成与KV缓存

**左侧——无缓存vs有缓存对比图（左右两栏）：**

```
═══════════════════════════════════════════════════════════════════
  ❌ 无缓存（每步重算全部）

  Step 1: 计算[START]的Q,K,V → 注意力 → 输出"我"

  Step 2: 重新计算[START,我]全部Q,K,V → 注意力 → 输出"爱"
           ↑ 重复计算START的K,V！

  Step 3: 重新计算[START,我,爱]全部Q,K,V → 注意力 → 输出"你"
           ↑ 又重复计算！

  计算量：O(T²)——每步都要重算之前所有词
═══════════════════════════════════════════════════════════════════

  ══════════════════════════════════════════════════════════════════
  ✅ 有缓存（KV Cache）

  Step 1: 计算[START]的K,V → 存入缓存
           Cache: {K: [k_START], V: [v_START]}

  Step 2: 只计算[我]的新K,V → 追加到缓存 → 注意力 → 输出"爱"
           Cache: {K: [k_START, k_我], V: [v_START, v_我]}
           ↑ 只算新词！

  Step 3: 只计算[爱]的新K,V → 追加到缓存 → 注意力 → 输出"你"
           Cache: {K: [k_START, k_我, k_爱], V: [v_START, v_我, v_爱]}
           ↑ 每步O(1)增量！

  计算量：O(T×d)——每步只算新词
═══════════════════════════════════════════════════════════════════
```

- 画Cache方框列出内容：{K: [k_START, k_我, k_爱], V: [v_START, v_我, v_爱]}
- 无缓存用红色标注重复计算，有缓存用绿色标注增量计算

**右侧——四段详细分析：**

① **自回归生成的工作机制：** 自回归（Autoregressive）意味着每个新词的生成依赖于之前所有已生成的词。具体过程：(1) 编码器处理输入"I love you"，输出编码向量并缓存；(2) 解码器输入[START]，通过掩码自注意力+交叉注意力+FFN，输出第一个词"我"（概率0.85）；(3) 将"我"追加到输入：[START, 我]，解码器输出"爱"（概率0.78）；(4) 重复直到输出"<END>"标记。整个过程严格串行，无法并行化。

② **无缓存的巨大浪费：** 在没有KV缓存的情况下，每生成一个新词，都需要将整个已生成序列重新送入解码器。生成第100个词时，需要重新计算前99个词的Q、K、V向量和注意力，而这些计算在第1-99步已经做过了。总计算量是O(T²)，其中T是序列长度——这意味着生成长度翻倍，计算量增加4倍。

③ **KV缓存=会议纪要：** KV缓存的核心思想是：每一步只计算新词的K和V向量，然后将它们追加到缓存中。做注意力计算时，新词的Q与缓存中的所有K做点积（而不是重新计算所有词的K）。这样每步的计算量是O(d×T)（d是维度，T是当前缓存长度），而非O(T×d×T)。这就像开会——不需要每次讨论都重新介绍所有背景，只需要查阅之前的会议纪要，只讨论新增的内容。

④ **KV缓存的代价：内存消耗。** 每一步都要缓存所有已生成词的K和V向量。对于12层、64头、128维的模型，生成100个词的缓存大小约为：12层 × 100词 × 2(K+V) × 64头 × 128维 × 2字节(float16) = 约375MB。生成长文本时，KV缓存可能成为内存瓶颈。GPT-4等大模型的KV缓存可以达到数GB。

**右侧——两段总结分析：**

**实际应用中的组合策略：** ChatGPT通常使用Temperature=0.7 + Top-P=0.9的组合。Temperature=0.7使分布适度尖锐（不会太随机），Top-P=0.9排除低质量候选。同时还会使用重复惩罚（Repetition Penalty）来避免生成循环内容。GPT-4在写代码时建议Temperature=0.2（需要精确），在创意写作时建议Temperature=0.7-1.0（需要多样性）。

**为什么不能总是选概率最高的词：** 贪心搜索看似最优（每步选最好的），但可能错过全局最优。例如："我喜欢在公园里___"，贪心选"散步"(p=0.3)，但"跑步"(p=0.25)+"之后喝杯咖啡"(p=0.4)的整体概率更高。束搜索部分解决了这个问题，但在开放域对话中，"最优"本身就不存在——我们需要的是多样性和创造性，而非单一最优解。

**右侧——三段详细分析：**

① **GPT系列（仅解码器）——生成之王：** 核心思想是"预测下一个词"。GPT系列只保留Transformer的解码器部分，去掉了编码器和交叉注意力。训练方式是自回归语言建模：给定前面的文本，预测下一个词。ChatGPT的三步训练流程：(1)预训练——在海量文本上预测下一个词（GPT-3训练了3000亿token）；(2)指令微调——在人工标注的指令-回复对上微调，让模型学会理解人类指令；(3)RLHF（人类反馈强化学习）——用人类偏好数据训练奖励模型，再用PPO算法优化生成策略。代表模型：GPT-3(1750亿参数)、GPT-4(约1.8万亿参数，传闻)、LLaMA(7B-65B)、Qwen(7B-72B)、GLM-4。

② **BERT系列（仅编码器）——理解之王：** 核心思想是"掩码语言建模"(MLM)。随机遮盖输入中15%的token，让模型根据上下文预测被遮盖的内容。例如，原文"我_svg北京"，模型需要预测_svg是"在"。BERT还使用"下一句预测"(NSP)任务来学习句子间的关系。由于没有解码器，BERT不能直接生成文本，但极其擅长文本理解任务：分类、问答、命名实体识别、语义相似度等。Google搜索从2019年开始使用BERT来理解用户查询。代表模型：BERT-base(1.1亿)、BERT-large(3.4亿)、RoBERTa、DeBERTa。

③ **T5系列（完整架构）——转换之王：** 核心思想是"将所有NLP任务统一为文本到文本(Text-to-Text)格式"。翻译任务：输入"translate English to German: That is good"→输出"Das ist gut"。摘要任务：输入"summarize: [长文章]"→输出"[摘要]"。分类任务：输入"classify: [文本]"→输出"positive/negative"。这种统一的框架极大地简化了NLP系统的设计。Google Translate基于T5的变体mT5。代表模型：T5-Small(6000万)到T5-XXL(110亿)。

**下方——三段详细分析：**

① **规模爆发：从6500万到万亿参数。** 2017年原始Transformer只有6500万参数。到2023年，GPT-4据估计有约1.8万亿参数（通过混合专家架构MoE实现），增长了约3万倍。架构本身几乎没有变化——核心改进是：(1)层数从6增加到80+；(2)嵌入维度从512增加到8192+；(3)注意力头数从8增加到128；(4)训练数据从450万句对增加到数万亿token。Scaling Law（Kaplan et al., 2020）揭示了：模型性能与参数量、数据量、计算量的幂律关系，为"大力出奇迹"提供了理论支撑。

② **超越语言：Transformer征服了所有模态。** 2020年，ViT（Vision Transformer）将图像切成16×16的小块（patch），每个块当作一个"词"，直接用标准Transformer处理图像分类任务——在ImageNet上达到了与卷积神经网络相当甚至更好的效果。2022年，OpenAI的Whisper使用Encoder-Decoder Transformer做语音识别，支持99种语言的翻译和转录。2023年，GPT-4V和Gemini实现了视觉-语言多模态理解，可以同时处理图片和文本。2024年，Sora使用视频扩散Transformer生成高质量视频。AlphaFold2使用Transformer的注意力机制预测蛋白质三维结构，准确率达到实验水平。

③ **核心洞察：Transformer是通用信息处理框架。** Transformer的成功揭示了一个深刻的事实：只要能将数据表示为序列（或切块后表示为序列），Transformer就能处理。无论是自然语言、图像像素、语音频谱、蛋白质氨基酸序列、还是时间序列数据，Transformer的核心机制——自注意力——都能捕捉数据内部的依赖关系。这就是为什么Transformer从一个机器翻译模型，演变成了"AI的通用引擎"。

**右侧——五段详细总结（每段对应一个要点）：**

① **Transformer的本质是让信息自由流动：** 自注意力机制使每个词都能与序列中的所有其他词直接交流，无需经过层层传递。这一核心设计同时解决了RNN的三大问题：并行化（所有词同时处理）解决顺序处理瓶颈，直接连接（任意词对直接交互）解决长距离遗忘，不压缩信息（每个词保持独立的表示）解决信息瓶颈。这是Transformer对深度学习最重要的贡献。

② **编码器理解，解码器生成：** 编码器像编辑部——深度分析输入文本，提取丰富的语义信息，每层都在"交流+提炼"。解码器像翻译部——在编码器理解的基础上，结合掩码自注意力和交叉注意力，逐步生成输出序列。两者通过K、V向量连接，实现了"理解→生成"的优雅分工。

③ **训练是开卷考试，推理是闭卷考试：** 训练时使用Teacher Forcing，解码器可以看到完整的目标序列，所有位置并行计算，效率极高。推理时必须自回归生成，逐词串行，前一步的输出是后一步的输入。两者的核心矛盾是"暴露偏差"——训练时看正确答案，推理时看自己的输出。KV缓存是缓解推理效率瓶颈的关键技术。

④ **三大家族各有所长：** GPT（仅解码器）擅长生成→ChatGPT/通义千问/GLM；BERT（仅编码器）擅长理解→Google搜索/文本分类；T5（完整架构）擅长转换→Google Translate/多任务。三者共享Transformer的DNA，却走上了完全不同的职业道路，各自统治了NLP的一个子领域。

⑤ **从一篇论文到AI时代的基石：** 2017年的Transformer论文最初只是为机器翻译设计的。7年后，它已经成为几乎所有AI系统的底层架构——从ChatGPT到AlphaFold2，从Sora到Whisper。参数量从6500万增长到万亿级，但核心机制从未改变。Transformer不仅仅是一个模型架构，它代表了一种全新的信息处理范式——让每个元素都能自由地与所有其他元素交流。这是AI时代的基石。
