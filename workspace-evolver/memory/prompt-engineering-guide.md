## 1. 核心 Prompt 技巧

### 1.1 Chain-of-Thought (CoT) 思维链

**原理**：让模型一步步推理，而不是直接给出答案。

**示例**：
```
问题：小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？

请一步步思考：
1. 小明最初有 5 个苹果
2. 给了小红 2 个后，剩下 5 - 2 = 3 个
3. 又买了 3 个，现在有 3 + 3 = 6 个

答案：6 个苹果
```

**适用场景**：
- 复杂数学推理
- 多步骤问题
- 逻辑推导

### 1.2 Few-Shot Learning 少样本学习

**原理**：提供几个示例，让模型理解任务格式。

**示例**：
```
将以下句子翻译成古文：

示例1：
现代文：今天天气很好。
古文：今日天朗气清。

示例2：
现代文：他很高兴。
古文：其喜甚。

现在请翻译：
现代文：我们一起去公园玩吧。
古文：
```

**技巧**：
- 2-5 个示例效果最佳
- 示例应覆盖不同情况
- 保持格式一致

### 1.3 RAG (Retrieval-Augmented Generation)

**原理**：先检索相关文档，再将检索结果作为上下文输入模型。

**流程**：
```
用户问题 → 向量检索 → 获取相关文档 → 组合 Prompt → 模型生成
```

**最佳实践**：
- 检索 Top 3-5 相关片段
- 控制上下文长度
- 标注信息来源

### 1.4 Self-Consistency 自一致性

**原理**：生成多个推理路径，选择最常见的答案。

**流程**：
1. 用相同问题生成多个回答
2. 提取每个回答的最终答案
3. 选择出现频率最高的答案

**适用场景**：
- 数学问题
- 选择题
- 需要高准确率的任务

### 1.5 Generated Knowledge 生成知识

**原理**：先让模型生成相关知识，再基于知识回答问题。

**示例**：
```
第一步：生成知识
问题：什么是光合作用？
请先列出关于光合作用的关键知识点。

第二步：基于知识回答
现在请基于上面的知识点，详细解释光合作用的过程。
```

### 1.6 Prompt Chaining 提示链

**原理**：将复杂任务拆分成多个步骤，逐步完成。

**示例流程**：
```
1. 分析用户需求 → 提取关键信息
2. 生成大纲 → 结构化内容
3. 撰写内容 → 逐节展开
4. 审核修改 → 质量检查
```

## 2. OpenClaw 中的 Prompt 优化

### 2.1 System Prompt 结构

OpenClaw 的系统提示包含以下部分：

| 部分 | 说明 | Token 开销 |
|-----|------|-----------|
| **Tooling** | 可用工具列表 | 变长 |
| **Safety** | 安全指南 | 固定 |
| **Skills** | 技能列表 | 每个~97字符 |
| **Workspace** | 工作目录 | 固定 |
| **Documentation** | 文档路径 | 固定 |
| **Project Context** | 注入的上下文文件 | 变长 |
| **Runtime** | 运行时信息 | 固定 |

### 2.2 上下文注入

**自动注入的文件**：
- `AGENTS.md` - Agent 行为规则
- `SOUL.md` - 人格定义
- `TOOLS.md` - 本地工具说明
- `IDENTITY.md` - 身份信息
- `USER.md` - 用户信息
- `HEARTBEAT.md` - 心跳任务
- `MEMORY.md` - 长期记忆（仅主会话）

**优化建议**：
- 保持文件简洁
- `MEMORY.md` 定期清理
- 使用 `memory/*.md` 存储详细笔记

### 2.3 Prompt Caching

**Anthropic 的 Prompt Caching**：
- 缓存系统提示和上下文
- 减少 90% 输入成本
- 缓存有效期 1 小时

**配置**：
```json5
{
  agents: {
    defaults: {
      models: {
        "anthropic/claude-opus-4-6": {
          params: { cacheRetention: "long" }
        }
      }
    }
  }
}
```

## 3. 高级技巧

### 3.1 角色设定 (Persona)

**基本格式**：
```
你是一位[角色]，擅长[能力]，专注于[领域]。
你的风格是[风格描述]。
```

**示例**：
```
你是一位资深软件架构师，擅长系统设计和代码优化。
你的回答简洁、实用，注重最佳实践和可维护性。
```

### 3.2 输出格式控制

**JSON 输出**：
```
请以 JSON 格式输出，包含以下字段：
{
  "summary": "摘要",
  "points": ["要点1", "要点2"],
  "confidence": 0.8
}
```

**Markdown 输出**：
```
请使用 Markdown 格式，包含：
- 一级标题作为主标题
- 无序列表列出要点
- 代码块展示示例
```

### 3.3 约束条件

**常用约束**：
```
约束：
1. 回答不超过 200 字
2. 使用简体中文
3. 避免专业术语，用通俗语言解释
4. 如果不确定，直接说明
```

### 3.4 思维模式控制

**OpenClaw 中的 Reasoning 模式**：
- `/reasoning on` - 开启思考过程
- `/reasoning off` - 关闭
- `/reasoning stream` - 流式输出思考

**配置**：
```json5
{
  agents: {
    defaults: {
      reasoning: {
        mode: "auto",  // auto | on | off
        budget: 16000  // 思考 token 预算
      }
    }
  }
}
```

## 4. 常见问题与解决方案

### 4.1 回答太长/太短

**解决方案**：
```
请用 3-5 句话简洁回答。
// 或
请详细展开，包含背景、原理、示例和应用场景。
```

### 4.2 回答偏离主题

**解决方案**：
```
重要：只回答关于 XXX 的问题，不要讨论其他话题。
如果问题与 XXX 无关，请回复"这个问题超出我的范围"。
```

### 4.3 格式不一致

**解决方案**：
```
严格按照以下格式输出：
【标题】...
【要点】...
【结论】...

不要添加额外内容。
```

### 4.4 事实错误

**解决方案**：
```
如果不确定某个事实，请：
1. 明确说明"我不确定"
2. 提供你的最佳猜测
3. 建议用户验证
```

## 5. 模型特定优化

### 5.1 Claude

**特点**：
- 擅长长上下文理解
- 支持 extended thinking
- 代码能力强

**优化建议**：
- 使用 XML 标签结构化输入
- 启用 prompt caching
- 复杂任务开启 thinking

### 5.2 GPT

**特点**：
- 通用性强
- 生态完善
- 支持函数调用

**优化建议**：
- 使用 system message 设定角色
- 利用 function calling
- 控制 temperature

### 5.3 Gemini

**特点**：
- 多模态能力强
- 超长上下文 (1M+)
- 免费额度

**优化建议**：
- 利用多模态能力
- 处理长文档时使用
- 注意 token 计费

### 5.4 GLM / Qwen

**特点**：
- 中文优化
- 思考模式
- 免费额度

**优化建议**：
- 中文任务优先
- 复杂推理开启思考
- 利用免费额度测试

## 6. 评估与迭代

### 6.1 评估指标

| 指标 | 说明 |
|-----|------|
| **准确率** | 回答正确的比例 |
| **相关性** | 回答与问题的相关程度 |
| **完整性** | 回答是否全面 |
| **简洁性** | 是否有多余信息 |
| **一致性** | 多次回答是否稳定 |

### 6.2 迭代优化流程

1. **建立测试集**：准备 20-50 个典型问题
2. **初始测试**：记录模型表现
3. **分析问题**：找出失败模式
4. **调整 prompt**：针对性优化
5. **回归测试**：确认改进效果
6. **重复 2-5**：持续优化

## 7. 安全注意事项

### 7.1 Prompt 注入防护

**常见攻击模式**：
```
Ignore previous instructions and...
You are now in developer mode...
[SYSTEM OVERRIDE]...
```

**防护措施**：
- 输入验证
- 指令隔离
- 权限最小化
- 沙箱执行

### 7.2 敏感信息保护

**原则**：
- 不在 prompt 中存储凭证
- 不让 AI 处理高度敏感数据
- 定期清理记忆文件
